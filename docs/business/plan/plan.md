## 1. 문제인식 (Problem)

### 1-1. 창업 아이템 배경 및 필요성

#### 가. 창업 동기 — "새벽 3시, 나는 내 아이가 왜 우는지 몰랐다"

2026년 2월, 첫 아들이 태어났다. 창업자는 5년차 Android·온디바이스 ML 개발자이기 이전에, **지금 이 순간 영아를 키우고 있는 보호자 당사자**다. 본 사업은 시장 조사가 아니라, 창업자 자신이 매일 밤 겪는 문제에서 출발했다.

**가장 힘든 것은 밤이었다.** 새벽에 아이가 울기 시작하면 — 배가 고픈 건지, 졸린 건지, 어딘가 불편한 건지 알 수 없었다. 원인을 모른 채 수유를 시도하고, 안아 흔들고, 기저귀를 확인하며 한참을 헤맸다. *"왜 우는지만 알아도 절반은 해결될 텐데"* 라는 생각이 매일 밤 반복됐다.

**기록도 문제였다.** 시중 수유 앱 대부분은 *"수유 버튼만 누르면 기록"* 되는 구조다. 한 손으로 아이를 안은 채 버튼을 누르면, 몇 ml를 먹였는지·분유인지 착유(모유)인지조차 담기지 않은 채 **일단 기록부터 되고**, 세세한 내용은 "나중에 입력"하라는 UX였다. 그러나 나중에 수유량을 늘렸을 때, 젖병을 바꿨을 때 그 기록을 **수정하는 일이 지나치게 번거로웠다.** 결국 기록은 부정확해졌고, **부정확한 기록 위에서는 어떤 패턴도 읽어낼 수 없었다.**

그리고 곧 깨달았다 — **나만 이렇게 헤매는 게 아니었다.** 수많은 부모가 이미 ChatGPT·Gemini에 *"신생아가 왜 우는지", "언제 재워야 하는지"* 를 묻고, 블로그에 그 활용법을 공유하고 있었다([브런치 "이방인 엄마의 ChatGPT 활용법"](https://brunch.co.kr/@hyeonjipark/34) · [코지맘 "엄마를 위한 ChatGPT 사용법"](https://cozymom.kr/%EC%97%84%EB%A7%88%EB%A5%BC-%EC%9C%84%ED%95%9C-chatgpt-%EC%82%AC%EC%9A%A9%EB%B2%95-%EC%99%84%EC%A0%84-%EC%A0%95%EB%A6%AC/) 등). 범용 챗봇에 의지할 만큼 **수요는 분명했지만**, 그 챗봇들에는 우리 아이의 **일지 데이터도, 울음소리도, 가족 공유도** 없었다. 누적 1,000만 다운로드 앱을 만든 5년차 온디바이스 ML 개발자로서, 나는 **"육아에 특화된" 그 도구를 직접 만들기로 했다.** **냠냠쿨쿨은 그렇게 시작됐다.**

#### 나. 문제 정의 — 기존 육아앱은 "기록"만 하고 "판단"은 부모 몫으로 남긴다

창업자의 경험은 개인의 불편이 아니라 **한국 영아 양육의 구조적 문제**와 정확히 맞닿아 있다. 문제는 세 겹으로 정의된다.

| 층위 | 문제 | 창업자 경험과의 연결 |
|---|---|---|
| **① 부모의 일상 (정서)** | "왜 울지? 지금 재워야 하나? 다음 수유는 언제?"를 하루 수십 번 판단해야 하는데 **근거가 없다 → 만성적 불확실성과 불안.** | 새벽 울음 원인을 몰라 헤맨 경험 |
| **② 도구의 한계 (구조)** | 기존 육아앱은 전부 **"기록(과거)" 도구**에 머물러, 부모가 진짜 원하는 **"예측(미래)·해석(지금의 의미)"**을 주지 못한다. 입력조차 양손 터치 전제라 아기를 안고 있을 때 기록이 끊기고, 수정이 번거로워 데이터가 부정확해진다. | "버튼만 누르면 기록" UX의 실패, 수정 불편 |
| **③ 시대 변화 (사회)** | 양육 인력은 줄고(맞벌이 58.5%·여성고용 71.3%) 1자녀당 양육 강도는 폭증(월 150만원·누적 3.65억). 양육이 **가족 협업**으로 바뀌었는데 정보는 단톡방에 흩어져 단절. | 시장 데이터 — 이하 라.~아. 에서 상술 |

> **핵심 한 문장:**
> **"기존 육아앱은 부모가 *기록하게* 만들 뿐, 부모를 *대신해 판단해주지* 않는다."**

#### 다. 창업 목적 — 우리가 만들려는 변화

> 본 사업의 목적은 단순한 기록 앱이 아니라, **부모를 대신해 판단하고 먼저 알려주는 능동적 육아 동행자**를 만드는 것이다.

**미션:** 부모가 데이터를 직접 해석하는 부담 없이, **AI가 먼저 "다음에 무엇을·언제 해야 하는지"를 알려주는 능동적 육아 동행자가 된다.**

이를 통해 이루려는 변화는 세 가지다.

| 대상 | 목적 |
|---|---|
| **개인** | 양육의 불확실성을 줄여 부모의 **불안을 낮추고 양육 효능감을 높인다.** |
| **가족** | 양육을 외로운 단독 노동에서, **가족이 함께 분담하는 협업으로 전환한다.** |
| **사회** | 저출산 시대의 **양육 부담 경감**에 기여한다. |

> 슬로건 **"잘 먹고, 잘 자는 다정한 동행"** 은 이 목적의 압축이다 — *잘 먹고 잘 자도록(예측·분석) 곁에서 함께(가족 협업) 다정하게(부모의 불안 경감) 돕는다.*

> 2. **AI 예측 영역은 5개 앱 전체 미보유** — 베이비타임이 입력 효율·성장 분석, 쑥쑥찰칵이 사진, 베이비빌리가 커뮤니티, 마미톡이 통합 정보, FamilyAlbum이 사진·영상으로 각자 우위. **"다음 수유·수면 시간을 미리 알려주는" 영역은 5개 모두 비어 있음 → 우리 P1 차별 영역**.
> 3. **음성비서 핸즈프리는 한국 시장 미진입** — 5개 앱 모두 양손 터치 입력 전제. 우리 P3 + 음성비서 메커니즘은 시장 신규 진입.
> 4. **대용량 사진·영상 클라우드 저장은 베이비타임의 빈틈** — 베이비타임은 사진 함께 저장만 제공하고 클라우드 갤러리·영상 백업이 약함. 우리는 **Kakao Cloud Object Storage 기반 사진·영상 무제한** (Pro 구독 정당화). 베이비타임 평생 ₩8,800 일회성으로는 대용량 인프라 운영 불가.
> 5. **베이비타임은 가격 모델 자체로 LLM·AI 진입 불가** — 일회성 ₩8,800 결제로는 Gemini Flash 호출당 $0.0013 비용을 지속 부담 불가. **AI 차별은 구독 모델만 가능 → 우리만의 영역**.

### 1-2. 창업 아이템 목표 시장(고객) 분석

#### 가. 시장 규모 (TAM / SAM / SOM)

> **2가지 시각 동시 제시** — ① 사용자 도달 단위(개인) / ② 결제 단위(가족). 평가자가 보수적 결제 시장과 광고·viral 도달 시장을 모두 검토할 수 있도록 함.

| 구분 | 정의 | 산정 근거 | 규모 |
|---|---|---|---|
| **TAM** (도달 시장, 개인 단위) | 0~24개월 영아 양육에 직간접 참여하는 모든 개인 (부·모·양가 조부모 평균 4명) | 49만 가구 × 4명 | **약 196만 명** (광고·viral 도달 기준) |
| **TAM** (결제 시장, 가족 단위) | 0~24개월 영아 양육 가구 (Pro/Family Plan 결제 단위) | **약 49만 가구** (2024 출생아 23.8만 + 2025 25.45만, [통계청](https://www.korea.kr/briefing/policyBriefingView.do?newsId=156745912)) | **약 49만 가구** (보수적 결제 기준) |
| **SAM** (유효 시장) | 스마트폰 사용 100% × 디지털 육아 보조 도입 의향 60% (베이비빌리 200만·시장 CAGR 14.6%로 검증) | 49만 가구 × 60% | **약 29만 가구** (결제 기준) |
| **SOM** (3년 도달 목표) | SAM의 약 6% 침투 — 베이비빌리·쑥쑥찰칵 활성 MAU 비교 시 보수적 | 29만 × 6% | **약 1.8만 가족 / DAU 약 20K** |

> **비판적 자가 검증**: 일부 평가자는 TAM 196만 산정을 "양가 조부모 4중 카운트"로 부풀리기로 해석할 수 있음. 이에 본 사업계획은 결제 단위(가족) 기준 TAM 49만을 병기. 단 광고·viral 도달은 가족 멤버 단위로 발생하므로 196만 도달 시장도 의미가 있음 (조부모는 직접 결제자가 아니더라도 카카오 알림톡 viral·광고 노출자로서 기여).

> **검증 논리**: 글로벌 육아앱 시장이 2025 $1.06B → 2034 $5.5B로 CAGR 14.6% 성장 중([Global Growth Insights](https://www.globalgrowthinsights.com/market-reports/parenting-apps-market-102179)). 베이비빌리는 한국에서 누적 200만 다운로드, 쑥쑥찰칵은 누적 100만 다운로드(80% 가입 전환). 우리 SOM 1.8만 가족은 결제 가족 단위 기준 SAM의 6%이며, 시드~Growth 4년간 단계적 달성.

#### 나. 타겟 페르소나 (3종)

| 페르소나 | 인구통계 | 핵심 니즈 | 페인 매핑 |
|---|---|---|---|
| **A. 지영 (32세, 신생아 엄마)** | 출산 3주차, 육아휴직 중 | **다음 수유·수면 타이밍 안내** + 울음 원인 + 자동 기록 | P1·P2·P3 |
| **B. 민수 (35세, 직장인 아빠)** | 평일 낮 분리, 육아휴직 사용 후 복직 | 하루 요약 + 사진 공유 + 패턴 인사이트 | P4 (정보 단절) |
| **C. 영자 (62세, 외할머니)** | 평일 낮 돌봄, 복잡한 앱 부담 | 큰 글씨 + 간단 입력 + 부모와 실시간 공유 | P4 + 디지털 진입 장벽 |

#### 다. 시장 트렌드 3가지

1. **1자녀당 양육 강도 폭증 → "프리미엄 육아" 수용성**
   영유아 가구 월 양육비 150만 원, 누적 양육비 3.65억(전 세계 1위). 1자녀당 가전·식품·디지털 보조 지출 증가, 월 ₩4,900 SaaS 구독에 대한 지불 의향 충분 (Huckleberry 미국 부모는 ₩13~20K 지불).

2. **가족 양육의 분산화 (협업형 양육 시대)**
   자녀 있는 부부 맞벌이 58.5% (+11.3%p / 10년), 30대 여성 고용률 71.3% (+14.4%p), 남성 육아휴직 10.2% 첫 돌파·연간 60.7% 증가. 단일 보호자 앱 → **가족 그룹 협업 앱**으로 패러다임 전환.

3. **AI 네이티브 부모 — 부모는 이미 LLM으로 육아 중**
   AI 예측·보조 도구에 대한 수요는 가설이 아니라 이미 현실이다. 미국 Common Sense Media(2023) 기준 학령기 자녀 부모 약 **30%가 챗GPT를 사용**했고([Futurism](https://futurism.com/artificial-intelligence/parents-chatgpt-rear-children)), 캔자스대 Life Span Institute 연구(부모 116명, *Journal of Pediatric Psychology* 2024.10)에서는 부모들이 **영아 수면교육·영양 정보에서 챗GPT가 쓴 글을 전문가가 쓴 글보다 더 신뢰·정확하다고 평가**했다([AAU](https://www.aau.edu/research-scholarship/featured-research-topics/study-finds-parents-relying-chatgpt-health-guidance)). 한국도 **국민 3명 중 1명이 챗GPT를 사용**하며([시사저널](https://www.sisajournal.com/news/articleView.html?idxno=259072)), *"챗GPT를 육아에 활용하는 부모가 늘고 있다"*([디지털투데이](https://www.digitaltoday.co.kr/news/articleView.html?idxno=557014)). ACM CHI 2024는 영유아 부모의 챗GPT 육아 활용을 별도 연구 주제로 다룰 만큼 보편화된 현상이다.
   > **인사이트**: 부모는 이미 범용 LLM에 묻고 있지만, ChatGPT·Gemini에는 **우리 아이의 일지 데이터도, 울음 오디오도, 가족 공유도 없다.** 이 간극이 곧 **육아 특화 AI 앱의 명확한 수요**다 — 냠냠쿨쿨은 일지·온디바이스 울음 분류·가족 협업을 LLM과 결합해 그 간극을 채운다.

#### 나. 핵심 메커니즘 ① — AI 수유·수면 예측 흐름

```
[누적 일지 (수유·수면·배변)] ─┐
                              ├─→ [LLM 패턴 분석] ─→ [다음 윈도우 예측]
[아기 프로필 (개월·체중)]    ─┘                       │
                                                       ▼
                          "오늘 오후 5:30~6:00에 다음 수유 가능성 높음"
                          "21:00경 졸음 신호 예상 — 침실 조명 낮춰두세요"
```

**구현**:
- 누적 일지 7~14일치를 LLM 컨텍스트에 압축 (요약 토큰 + 패턴 추출)
- Gemini 2.5 Flash 기본 사용, 호출당 ~$0.0002
- 무료: 일 1회 자동 예측 / Pro: 무제한 + 가족별 푸시
- Huckleberry SweetSpot 대비 한국화 + 30~60% 저렴 + 가족 단위 공유

#### 다. 핵심 메커니즘 ② — 울음 AI 분석 2-tier 파이프라인

```
[오디오 10초]
       │
       ▼
┌──────────────────────────────────────┐
│ Tier 1: 자체 학습 음향 분류기        │ ← 온디바이스 (Core ML / TFLite)
│  - 4-mega 라벨 (배고픔·불편·통증·졸림)│   지연 < 0.5초, 비용 0
│  - is_cry 게이트키퍼                  │   오디오 디바이스 미이탈
└──────────────────────────────────────┘
       │ (텍스트 페이로드만 생성: 분류 결과 + 일지 요약)
       ▼
┌──────────────────────────────────────┐
│ Tier 2: 텍스트 LLM (종합 추론)       │ ← Gemini 2.5 Flash 기본
│  - 분류 신호 + 일지 신호 종합        │   지연 2~3초, 비용 ~$0.0002
│  - 1~3순위 원인 + 권장 행동 + 면책   │   모든 텍스트 LLM 동등 호환
└──────────────────────────────────────┘
       │
       ▼
[결과 카드 + 일지 자동 연결]
```

**3중 효과**:
1. **프라이버시**: 오디오는 디바이스를 벗어나지 않음
2. **BYOK 자유**: 텍스트만 사용 → OpenAI·Anthropic 등 오디오 미지원 모델도 동등 호환
3. **오프라인 가능**: 네트워크 차단 시 분류기 결과만으로 결과 카드 표시

#### 라. 핵심 메커니즘 ③ — 입력 UX 4가지 혁신

쑥쑥찰칵 사용자 실측 인터뷰에서 도출한 4대 페인을 다음 메커니즘으로 해결.

| 페인 (쑥쑥찰칵) | 해결 메커니즘 | 측정 가능 KPI |
|---|---|---|
| 시각 휠 번거로움 | 기본값 = 현재 시각 (1탭 확인) + 상대 시각 칩("방금/10분 전/30분 전") + 음성 자연어 | 시각 입력 **0~1탭** |
| 종류·양 선택지 부담 | 직전 입력 Smart Default + 자주 쓰는 양 칩(60·90·120·150·180ml) + 1탭 패턴 복제 | 종류·양 선택 **0~1탭** |
| 한 손 조작 어려움 | 핵심 버튼 하단 Thumb Zone + 60dp 터치 영역 + 좌·우 스와이프 + 음성비서(0손) | 한 손 조작 **100%** |
| 수면 시작/종료 두 번 누름 | 사후 일괄 입력("방금 깼어" 1탭) + 누적 패턴 기반 잠든 시각 자동 추정 | 수면 1건 **1탭** |

> **목표 KPI**: 쑥쑥찰칵 일지 1건 입력 20~30초 → **냠냠쿨쿨 3~5초** (또는 음성 한 문장).
