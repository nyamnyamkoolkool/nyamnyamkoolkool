### 2.1 UC-1.1 울음 분석 (10초 오디오)

## 4. 비용 최적화 전략

### 4.1 캐시 적극 활용
- **`promptHash` 기반 응답 캐시** ([data-model.md](../usecase/_specs/data-model.md) `llmResponseCache` 컬렉션)
- AI 팁: 동일 컨텍스트(같은 아기·같은 일) 캐시 적중률 ~50% 가능
- 울음 분석: 오디오 fingerprint 캐시 적중률 낮음 (~5%) — 큰 효과 X
- Claude 프롬프트 캐시: 시스템 프롬프트 + 일지 요약을 캐시 → 90% 절감

### 4.2 모델 다운그레이드 정책
- 야간(0~6시): Gemini Flash-Lite로 자동 다운그레이드 (50%+ 절감, 정확도 trade-off)
- 무료 사용자 4번째 분석부터: Gemini Flash-Lite
- 정확도 신뢰도가 낮은 결과는 사용자에게 "본 모델로 재분석" CTA

### 4.3 Gemma 4 폴백 비율 상향
- 운영사 키 무료 한도 도달 시 자동으로 Gemma 4 폴백
- 사용자 옵션: "데이터 절약 모드" → 항상 Gemma 4 우선
- 디바이스 모델 부족 시 가입 시점 백그라운드 다운로드

### 4.4 Batch API 활용
- AI 팁 생성: 24시간 지연 허용 → 배치 50% 할인
- 패턴 인사이트: 일 1회 새벽 배치 → 50% 할인
- 울음 분석: 실시간 필수, 배치 불가

### 4.5 BYOK 유도 = 비용 0 transfer
- Pro 구독 ($5/월) 가입 시 Gemini OAuth 연결 권장 → 운영사 부담 0
- 본 LLM 호출 비용 자체가 사용자 본인 Gemini Pro 구독에 포함 → 사실상 운영사 비용 0

## 5. LLM 비용 한도 관리 (`llmUsageQuota` 활용)

### 5.1 가족당 일/월 한도 (가족 멤버 8명 반영, 2026-05-20 갱신)
| 사용자 유형 | 운영사 키 일 한도 | 운영사 키 월 한도 |
|---|---|---|
| **무료** | **8건 분석 / 5건 AI 팁** | **240건 분석 / 150건 팁 / 패턴 월 4건** |
| Pro (운영사 키 사용) | 무제한 | (Fair Use Policy 적용) |
| Pro (Gemini OAuth) | 무제한 (사용자 본인 키) | — |
| BYOK | 무제한 | — |

### 5.2 한도 도달 시 흐름
1. `llmUsageQuota/{yearMonth}.requestCount` increment → 한도 비교
2. 초과 시 응답: "오늘의 무료 분석 한도를 사용했어요. Pro로 업그레이드하거나 내일 다시 시도해주세요"
3. Gemma 4 폴백 옵션 제공: "오프라인 분석으로 계속할까요?"
4. **"광고 1회 보고 1건 즉시 충전"** 옵션 검토 — 추가 광고 수익 + 무료 사용자 가치 증가 (A/B 필요)

### 5.3 알림 트리거
- Pro 구독자 한도 80% 도달 시 알림
- 운영사 무료 한도 90% 도달 시 운영자에게 일일 알림 (긴급 모니터링)
