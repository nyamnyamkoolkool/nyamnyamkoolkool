# UC-1 자체 분류기 학습 계획

> 처리 시나리오는 [pipeline.md](./pipeline.md), 사용자 시나리오는 [README.md](./README.md) 참조.
> 최종 갱신: 2026-05-22

본 문서는 [pipeline.md](./pipeline.md)의 **Tier 1 자체 음향 분류기**를 어떻게 만들고 운영할지를 정의한다. 운영하는 모델의 수명주기는 앱 수명주기와 다르므로 별도 문서로 분리.

---

## 1. 결정 배경

- 외부 사전학습 모델(DeepInfant V2 등) 그대로 채택하지 않는다.
  - 학습 데이터 비공개·라벨 신뢰성 미검증
  - 한국 영아·국내 가정 환경 잡음(에어컨·식기·TV 한국어)에 대한 적합성 미확인
  - 유지보수 의존성 리스크
- 우리는 분류기를 **자체 학습**하되, **외부 공개 데이터셋으로 cold start**하고 **사용자 환류로 점진 개선**한다.

---

## 2. 모델 목표 지표

| 단계 | 목표 |
|---|---|
| **v0.1 (cold start)** | 4-mega 클래스 top1 정확도 ≥ 60%, not_cry 식별 ≥ 90% |
| **v0.5 (3개월 후)** | top1 정확도 ≥ 70%, 한국 가정 잡음 환경에서 SNR 5dB 이상 안정 |
| **v1.0 (12개월 후)** | top1 정확도 ≥ 75%, 가족 그룹 개인화 모델 옵션 제공 |

**왜 70%대가 천장인가**: ChatterBaby(UCLA) 3분류 71.5%, 학술 컨센서스(2024 Nature)상 원인 분류 자체에 정보 이론적 한계 존재. 80%+ 주장 앱은 라벨 자기일치(self-consistent labeling) 가능성. **우리는 정확도보다 attribution과 학습 가능성에 베팅한다.**

## 3. 클래스 정의

### v0.1 (4-mega + not_cry)

| 클래스 | 정의 | 외부 데이터셋 매핑 |
|---|---|---|
| `cry_hunger` | 배고픔·수유 요구 신호 | Donate-a-Cry: `hungry` |
| `cry_discomfort` | 기저귀·온도·자세 등 신체 불편 | Donate-a-Cry: `discomfort`, `belly_pain`, `burping` 통합 |
| `cry_pain` | 통증·강한 스트레스 (ChatterBaby 정의 활용) | Donate-a-Cry: `belly_pain` 일부 + 자체 라벨 |
| `cry_sleepy` | 졸림·피곤 | Donate-a-Cry: `tired` |
| `not_cry` | 우는 소리 아님 (잡음·말소리·울음 아닌 영아 발화) | AudioSet/YAMNet 비-cry 클래스에서 샘플링 |

**왜 4 + 1인가**: 9분류는 confidence 분산이 커 사용자 가치 감소. 사용자가 실제로 다른 행동을 취하는 단위로 묶음(배고픔→수유, 불편→점검, 통증→관찰·내원, 졸림→재움).

### v0.5+ 확장 후보 (사용자 데이터 누적 후 결정)
- `cry_pain` 세분화: 일반 통증 vs 응급(고통 시그널)
- `cry_attention` (애착·외로움) — 데이터로 분리 가능성 확인 후
- 영아 vs 신생아 발달 단계별 모델 분기

## 4. 데이터 전략

### 4.1 외부 공개 데이터셋 (Cold Start)

| 데이터셋 | 샘플 | 라이센스 | 용도 |
|---|---|---|---|
| **Donate-a-Cry** | 457 | ODbL/DBCL | 1차 라벨 부트스트랩 (5분류 → 4-mega 매핑) |
| **AudioSet (non-cry 부분)** | 수만 (선별) | CC-BY | `not_cry` 클래스 음원 (가정 잡음, 영아 비-울음) |
| **ICSD** | 공개 | 공개 | 울음/비-울음 이진 검증 보조 |
| **Baby Chillanto** | 2,268 | 비공개 요청 | ⚠️ 병리 진단 데이터, 1차 후보에서 제외 (윤리 검토 별도) |
| **Ubenwa CryCeleb** | 786 영아 | **CC-BY-NC-ND** | ❌ 비상업 — 상용 앱 학습에 사용 불가 |

**라이센스 주의사항**: 외부 데이터셋 각각의 라이센스를 **모델 아티팩트의 라이센스에 attach**해야 함. 상업 사용 시 ODbL의 share-alike 영향 검토 필요(법무 자문).

### 4.2 자체 수집 (Active Loop)

[pipeline.md](./pipeline.md) §7의 `user_feedback`을 통해 환류 수집:

1. UC-1.1 step 6: 사용자가 분석 결과 카드에서 **"맞았어요/달랐어요(원인 선택)"** 입력
2. `consent_for_training=true` 동의 사용자만 대상
3. 오디오 클립(분류기에 들어간 원본) + 분류기 출력 + 사용자 라벨이 한 튜플로 저장
4. 저장소: Kakao Cloud Object Storage (사진과 동일 인프라), 별도 버킷·암호화
5. PII 마스킹: 아기 식별자는 가명 토큰, 가족 그룹 ID 비저장
6. 보관 기간: 학습 완료 후 12개월, 사용자가 언제든 삭제 요청 가능 (GDPR/개인정보보호법 대응)

**목표 환류량**: 가입자 1,000명 기준 월 ~500건 사용 동의 샘플 예상(보수적 추정). 6개월 ~3,000건이면 fine-tuning 가치 있음.

### 4.3 데이터 균형

- 클래스 불균형 강함(배고픔 다수, 통증 소수) → 클래스 가중치 + 데이터 증강
- 증강: pitch shift ±2 semitone, time stretch 0.9~1.1x, AWGN +5/10dB, 가정 환경 잡음 믹스(에어컨·식기·한국어 TV)
- **한국 가정 잡음 음원 자체 수집** 필요 — 별도 안내 동의 (아기 무관 환경음만)

## 5. 모델 아키텍처

### v0.1 (MVP)
- **베이스**: YAMNet embedding (Google, AudioSet 사전학습) → frozen
- **헤드**: 작은 CNN 또는 단일 FC 레이어 — 5-class softmax
- **이유**: YAMNet은 "baby cry"를 521 클래스 중 하나로 이미 학습 → 임베딩 공간이 우리 작업에 호의적
- **장점**: 데이터 적어도 fine-tuning 빠름, 모델 크기 작음 (TFLite < 5MB)

### v0.5+ (데이터 누적 후)
- 헤드 확장 또는 backbone 자체 fine-tuning
- 아키텍처 후보: MFCC + CNN-LSTM(ChatterBaby 식), 또는 Audio Spectrogram Transformer 경량 변형
- 단, **양자화 후 모바일 추론 50ms 이내** 제약

### 배포 포맷
- iOS: Core ML (`.mlmodel`)
- Android: TensorFlow Lite (`.tflite`)
- 변환: TF → TFLite는 표준, Core ML은 `coremltools` 사용
- **OTA 업데이트**: 모델 파일은 앱 번들이 아니라 **Firebase Hosting 또는 Storage에서 버전별 다운로드** — 앱 재배포 없이 모델 개선 배포 가능

## 6. 평가 방법

### 6.1 오프라인 평가 (개발 단계)

| 분할 | 출처 | 용도 |
|---|---|---|
| Train (70%) | 외부 데이터셋 + 자체 수집 | 학습 |
| Val (15%) | 외부 데이터셋 stratified split | 하이퍼파라미터 |
| Test (15%) | **한국 영아 자체 수집 only** | 최종 보고 |

**중요**: Test 분할에는 외부 데이터를 섞지 않는다. 외부 데이터셋의 라벨 신뢰성 자체가 의심 가는 상황에서, 외부 데이터로 평가하면 over-reporting 위험.

### 6.2 온라인 평가 (배포 후)

- 사용자 환류 라벨을 정답으로 가정한 **rolling accuracy** 측정
- 단, 사용자 라벨도 정답이 아님(부모도 100% 정확하지 않음) → 신뢰구간 표기
- A/B 테스트: 모델 v0.1 vs v0.2 → 사용자 만족도(맞았어요 비율) 차이

### 6.3 평가 지표

- **Macro-F1** (클래스 불균형 보정)
- **Top1 accuracy** (사용자 노출 기준)
- **Confusion matrix**: 어떤 쌍이 혼동되는지 — 통증/불편 혼동은 위험, 졸림/배고픔 혼동은 비교적 안전
- **SNR별 정확도 분해**: 5/10/15/20 dB 구간 — 실사용 환경 적합성

## 7. 베이스라인 비교

다음 베이스라인 대비 우리 모델이 어디서 개선하는지 보고:

| 베이스라인 | 비고 |
|---|---|
| **일지 휴리스틱 only** | "마지막 수유로부터 시간 경과" 룰베이스. 우리 분류기가 이걸 능가해야 가치 증명 |

| **DeepInfant V2** | 외부 모델 직접 비교. 우리 자체 학습이 가치 있는지 정당화 |
| **Donate-a-Cry SOTA 모델** | 학술 SOTA 대비 우리 모델 수준 |

## 8. 단계별 마일스톤

| 단계 | 기간 | 산출물 | Gate |
|---|---|---|---|
| **M1: 데이터 수집·정리** | 0~4주 | 외부 데이터셋 통합, 4-mega 라벨 매핑 스크립트 | 5,000+ 클립 정리 완료 |
| **M2: v0.1 학습** | 4~8주 | YAMNet + FC 헤드 모델, TFLite/CoreML 변환 | Test set top1 ≥ 60% |
| **M3: 통합 PoC** | 8~10주 | 앱 통합, [pipeline.md](./pipeline.md) E2E 동작 | 자체 50건 한국 영아 실측 ≥ 55% |
| **M4: 클로즈드 베타** | 10~16주 | 사용자 환류 수집 시작 | 동의 사용자 ≥ 100명, 환류 ≥ 500건 |
| **M5: v0.2 fine-tune** | 16~24주 | 환류 데이터 통합 모델 | top1 ≥ 65% (실사용 환경) |
| **M6: 공개 출시** | 24주~ | OTA 갱신 파이프라인 가동 | 사용자 적중률 ≥ 60% 안정 |

각 게이트 미통과 시 단계 재진입. M3 게이트 실패가 반복되면 **UC-1을 일지 휴리스틱 only로 강등** 결정 — 본 시나리오의 폐기 기준.

## 9. 법적·윤리적 고려사항

### 9.1 동의 (Consent)
- 아기 음성은 **민감 개인정보**에 준해 처리
- 학습용 업로드는 **별도 추가 동의** 필요 (가입 약관 동의와 분리)
- 동의 철회 시 학습 데이터에서 즉시 제거 가능한 추적 ID 필수

### 9.2 데이터 보호
- 업로드 시 TLS, 저장 시 서버 측 암호화
- 가족 그룹 ID·아기 식별자는 학습 파이프라인 진입 전 가명화
- 학습된 모델은 원본 오디오를 재구성할 수 없음 (단방향) — 모델 추출 공격 방어 정책 별도

### 9.3 편향 (Bias)
- 외부 공개 데이터셋은 **서구 영아 중심**일 가능성 → 한국 영아 데이터 누적 전까지는 편향 가능성 명시
- 보호자에게 "정확도는 데이터가 쌓일수록 좋아진다" 안내 (UC-1.1 A4와 일관)

### 9.4 책임
- 분류기 결과로 의료 결정 내리지 않도록 면책 문구 의무화 (UC-4.3 안전성과 일관)
- 응급 키워드 감지 시 "즉시 의료기관" 안내가 분류기 결과 위에 우선 표시

## 10. 미해결 사항

- **외부 데이터셋 상업적 사용 법무 검토**: Donate-a-Cry ODbL share-alike 조항이 모델 아티팩트에 미치는 영향
- **학습 인프라 위치**: 자체 GPU vs 클라우드(GCP Vertex AI, NCP, Kakao Cloud GPU) — 비용 비교 필요 ([_specs/cloud-cost-comparison.md](../_specs/cloud-cost-comparison.md) 갱신 사항)
- **개인화 모델 (v1.0+)**: 가족 그룹별 모델 분기 시 저장·배포 비용
- **모델 카드(Model Card) 게시 의무**: AI 투명성 차원에서 사용자에게 모델 한계 공개 방식
- **DeepInfant 등 외부 모델을 PoC 단계에서 임시 사용할지**: 자체 모델 v0.1 학습 전 PoC 구간에 DeepInfant CoreML을 일시 사용 → M3 게이트 직전 자체 모델로 교체 옵션
