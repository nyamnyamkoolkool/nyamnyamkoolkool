# UC-1. 울음소리 AI 분석

> **자체 학습 음향 분류기 (온디바이스)** + **텍스트 LLM (종합 추론)** 의 2-tier 파이프라인으로, 울음 신호를 누적 일지 컨텍스트와 결합해 추론한다.

상위: [메인 시나리오](../main-scenario.md)
관련 문서:
- 처리 시나리오 → [pipeline.md](./pipeline.md) (분류기·LLM 호출 순서, 입출력 스키마, 폴백)
- 분류기 학습 계획 → [training-plan.md](./training-plan.md) (전략·데이터·모델·마일스톤)
- 학습 실무 핸드북 → [datasets-and-training.md](./datasets-and-training.md) (데이터셋 카탈로그·환경 셋업·코드 예시·변환)

## 개요

**오디오를 멀티모달 LLM에 직접 던지지 않는다.** 학술 컨센서스(2024 Nature)상 울음 의도 분류 자체에 정보 이론적 한계가 있고, 일반 멀티모달 LLM은 비음성 분류에 공식 면책되어 있다.

대신 2-tier 구성:

| Tier | 컴포넌트 | 입력 | 출력 | 위치 |
|---|---|---|---|---|
| **1** | 자체 학습 음향 분류기 | 오디오 (PCM 16kHz) | 4-mega 라벨 확률 + SNR | 온디바이스 |
| **2** | 텍스트 LLM | 분류기 라벨 + 일지 + 프로필 (텍스트만) | 자연어 추론 + 권장 행동 | 네트워크 |

**효과**:
- LLM 호출에 오디오를 보내지 않으므로 **모든 텍스트 LLM 프로바이더가 동등 사용 가능** (Claude/OpenAI/Anthropic 오디오 미지원 문제 해결)
- 분류기는 온디바이스 — 오프라인에서도 1차 결과 표시 가능
- 분류기·LLM 출력이 분리되어 attribution과 검증이 가능

**LLM 프로바이더** (Tier 2 한정, 모두 텍스트 호출):
- 기본: Gemini 2.5 Flash (Google AI Studio)
- 유료 사용자: Gemini 컨슈머 OAuth 연결 시 최신 Gemini 모델
- BYOK: OpenAI / Anthropic (텍스트 호출만 사용 → 모델 제약 무관)
- 오프라인 LLM 폴백: 텍스트 소형 모델 또는 룰베이스 (멀티모달 Gemma 불필요)
- 상세: [UC-4.4](../UC-4-baby-info-settings/README.md#uc-44-ai-프로바이더-선택-및-관리)

## UC-1.1 실시간 울음소리 분석

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 아기 울음 원인을 컨텍스트와 함께 추론 |
| 사전조건 | 앱 로그인, 마이크 권한, 아기 프로필 등록 |
| 사후조건 | 분석 결과 저장, 후속 일지 작성 연결 가능 |

**주 시나리오**
1. 홈 화면 **[울음 분석]** 탭
2. **녹음** (기본 10초, **[더 녹음]** 으로 최대 30초)
   - 5초 미만에서 종료 시도 시 "조금 더 녹음해야 정확해요" 안내 후 자동 연장
   - 녹음 길이 하한은 분류기 모델 사양에 따라 조정 가능 (자세한 사양은 [pipeline.md](./pipeline.md) §3)
3. **Tier 1 (온디바이스 분류기 실행, < 0.5초)**:
   - 4-mega 라벨 확률 + SNR 산출
   - `is_cry=false` → 안내 후 종료 (LLM 호출 안 함)
4. **Tier 2 (텍스트 LLM 호출, 2~5초)** — 활성 프로바이더에 다음 텍스트 페이로드 전송:
   - 분류기 출력 라벨·확률
   - 아기 프로필 (개월수, 특이사항/알러지 — 가명화)
   - 직전 24시간 일지 요약 (마지막 수유 시각·양, 최근 수면, 최근 배변)
   - 환경 컨텍스트 (시각, 날씨 — 선택)
5. LLM 구조화 응답:
   - 1~3순위 원인 후보 (confidence band: high/medium/low)
   - 분류기 신호와 일지 신호 중 어느 쪽이 더 강한지 명시
   - 추론 근거 (자연어), 권장 행동, 안전 면책 문구
6. 결과 카드 표시 (예시)
   > **1순위: 배고픔 — 가능성 높음**
   > 음향 신호: 배고픔 패턴 (62%), 일지 신호: 마지막 수유 2시간 35분 전 (강함)
   > 두 신호가 같은 방향을 가리킵니다. 평소 이 시간대 수유 간격을 넘었어요.
   > → **추천: 수유 준비 / 90~120ml 분유**
   > 참고용이며 의학적 진단이 아닙니다.
   >
   > 2순위: 불편함 — 가능성 중간 (기저귀 마지막 확인 4시간 전)
7. **[일지에 기록]** → 후속 행동을 바로 입력
8. **[맞았어요/달랐어요]** 피드백 → 사용자 동의 시 [training-plan.md](./training-plan.md) §4.2의 환류 데이터로 사용

**대안 흐름**
- **A1. 주변 소음 큼 (SNR < 5dB)**: "주변이 시끄러워요. 가까이서 다시 녹음해주세요" 안내 — Tier 1에서 결정, LLM 호출 안 함
- **A2. 마이크 권한 거부**: 권한 설정 화면 유도
- **A3. 울음 미감지 (`is_cry=false`)**: Tier 1이 거부 — "우는 소리가 감지되지 않았어요" 안내
- **A4. 일지 데이터 부족(신규)**: 분류기 결과 + 프로필만으로 LLM 추론. "기록이 쌓일수록 정확해져요" 안내
- **A5. 활성 프로바이더 호출 실패**: 등록된 다른 프로바이더로 자동 폴백 (사용자 동의 옵션)
- **A6. 분류기 신뢰도 낮음 (top1_conf < 0.40)**: 분류기 라벨을 `ambiguous`로 LLM에 전달 → LLM이 일지 신호만으로 추론

**예외 흐름**
- **E1. 네트워크 오류 / LLM 호출 실패**: **Tier 1 분류기 결과만으로 결과 카드 표시** ("일지 기반 종합 추론은 온라인 복귀 시"). 멀티모달 Gemma 폴백 필요 없음.
- **E2. LLM 응답 지연 (>10초)**: 진행 표시 후 타임아웃 시 재시도 → 실패 시 E1과 동일 처리
- **E3. 분류기 모델 미설치 (앱 첫 실행)**: 백그라운드 다운로드 안내. 다운로드 전에는 일지-only LLM 호출
- **E4. 모호 응답**: 폴백 문구("판단이 어려워요. 가까운 시각의 수유/기저귀를 먼저 확인해보세요") + 사용자 직접 라벨링 옵션

**프라이버시·비용**
- **오디오는 디바이스를 벗어나지 않는다.** Tier 1 분류기는 온디바이스. LLM에는 텍스트만 전송
- 학습용 오디오 업로드는 **별도 추가 동의** (가입 약관과 분리, [training-plan.md](./training-plan.md) §9.1)
- PII 마스킹 (아기 실명 → 가명 토큰)
- 동일 페이로드 해시 5분 캐싱으로 LLM 비용 절감 (분류기는 추가 비용 0)
- LLM 호출 비용: 텍스트 토큰만 — 호출당 ~$0.0002 (기존 멀티모달 호출 대비 ~95% 절감)

## UC-1.2 울음 패턴 히스토리 조회

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 누적 울음 분석 결과로 아기 울음 패턴 이해 |

**주 시나리오**
1. **[울음 기록]** 탭 진입
2. 시간대별/카테고리별 통계 차트 — 예) "최근 7일간 17~19시에 '배고픔' 울음 집중"
3. **[패턴 분석]** → LLM이 누적 결과 + 일지로 인사이트 생성
   - 예) "저녁 수유를 30분 앞당겨보는 것을 추천드려요. 최근 5일 중 4일이 18시 직전에 배고픔 울음으로 시작됐어요."
4. 생성된 인사이트는 [UC-4.3 AI 맞춤 팁](../UC-4-baby-info-settings/README.md#uc-43-ai-맞춤-팁-제공)에 통합되어 홈 화면에 표시됨

## 비기능·운영 사항

- **녹음 조건**: 분류기 모델 사양에 따라 5~30초 (현재 기본 10초). 자세한 사양은 [pipeline.md](./pipeline.md) §3.
- **오프라인 정책**: **Tier 1 분류기 단독 결과 표시**. 텍스트 LLM은 폴백 룰베이스 또는 소형 텍스트 모델 (멀티모달 Gemma 불필요).
- **응답 시간 목표**: 끝-끝 < 4초 (Tier 1 < 0.5s + LLM 2~3s). 분류기 단독 모드는 < 1초.
- **호출 비용**: Tier 2 한정 ~$0.0002/호출 (텍스트 토큰만)
- **검증 게이트**: 사용자 1순위 적중률 ≥ 60% (UC-1.0). 미통과 시 UC-1 강등 재검토 — [training-plan.md](./training-plan.md) §8.

## UC-1.0 기술 가설 검증 (Pre-MVP)

**가설**: "온디바이스 분류기 + 일지 컨텍스트 + 텍스트 LLM 추론" 조합이 부모가 수용 가능한 원인 추정을 제공한다.

**검증 지표**: 사용자 1순위 적중률(사후 "맞았어요" 비율) ≥ 60%
- 학술 컨센서스(2024 Nature)·UCLA ChatterBaby 71.5% 등을 고려한 보수적 목표
- < 40% 또는 사용자 NPS < 0이면 **UC-1 기능 자체 재설계** (오디오 제거, 일지 휴리스틱 only로 강등)

**검증 방법** ([training-plan.md](./training-plan.md) §6 참조)
1. 자체 수집 한국 영아 데이터 100건 — 외부 공개 데이터셋은 라벨 신뢰성 문제 있어 평가 부적합
2. 일지-only / 분류기-only / 결합 3-way 비교
3. 단순 룰베이스("마지막 수유 후 X시간 경과")와 비교
