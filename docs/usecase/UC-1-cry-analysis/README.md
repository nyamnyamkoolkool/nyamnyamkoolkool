# UC-1. 울음소리 AI 분석

> 멀티모달 LLM(오디오 입력)을 활용해 아기 울음의 원인을 누적 일지 컨텍스트와 결합하여 추론한다.

상위: [메인 시나리오](../main-scenario.md)

---

## 개요

단일 분류 딥러닝 모델 대신 **멀티모달 LLM**을 사용한다.
울음소리 + 아기 프로필 + 직전 24시간 일지 + 환경 컨텍스트를 통합 입력으로 전달하여 자연어 추론 결과를 받는다.

- **기본 모델**: Gemini 2.5 Flash (Google AI Studio)
- **유료 사용자**: Gemini 컨슈머 OAuth 연결 시 최신 Gemini 모델 사용
- **대체 가능**: OpenAI / Anthropic (BYOK)
- **오프라인 폴백**: Gemma 4 (온디바이스)
- 프로바이더 선택/관리 상세: [UC-4.4](../UC-4-baby-info-settings/README.md#uc-44-ai-프로바이더-선택-및-관리)

## UC-1.1 실시간 울음소리 분석

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 아기 울음 원인을 컨텍스트와 함께 추론 |
| 사전조건 | 앱 로그인, 마이크 권한, 아기 프로필 등록 |
| 사후조건 | 분석 결과 저장, 후속 일지 작성 연결 가능 |

**주 시나리오**
1. 홈 화면 **[울음 분석]** 탭
2. **최소 10초 이상** 녹음 (기본 10초, **[더 녹음]** 으로 최대 30초)
   - 10초 미만에서 종료 시도 시 "조금 더 녹음해야 정확한 분석이 가능해요" 안내 후 자동 연장
3. 활성 프로바이더(기본 Gemini 2.5 Flash, 오프라인 시 Gemma 4)로 다음 전송:
   - 오디오 클립
   - 아기 프로필 (개월수, 특이사항/알러지)
   - 직전 24시간 일지 요약 (마지막 수유 시각·양, 최근 수면, 최근 배변)
   - 환경 컨텍스트 (시각, 날씨 — 선택)
4. LLM 구조화 응답:
   - 1~3순위 원인 후보 (확률)
   - 추론 근거 (자연어)
   - 권장 행동
5. 결과 카드 표시 (예시)
   > **1순위: 배고픔 (높음)**
   > 마지막 수유 2시간 35분 전, 평소 이 시간대 수유 간격을 넘었어요. 울음 패턴도 짧고 반복적이라 배고픔 신호와 일치합니다.
   > → **추천: 수유 준비 / 90~120ml 분유**
   >
   > 2순위: 불편함 (중간) — 기저귀 마지막 확인 4시간 전
6. **[일지에 기록]** → 후속 행동을 바로 입력. 선택된 행동은 다음 분석의 피드백 신호로 사용

**대안 흐름**
- **A1. 주변 소음 큼**: LLM이 신뢰도 저하 명시 시 "조용한 환경에서 재시도" 안내
- **A2. 마이크 권한 거부**: 권한 설정 화면 유도
- **A3. 울음 미감지**: 10초 후 유의미한 신호 없으면 종료, 재시도 옵션
- **A4. 일지 데이터 부족(신규)**: 오디오+프로필만으로 분석, "기록이 쌓일수록 정확해져요" 안내
- **A5. 활성 프로바이더 호출 실패**: 등록된 다른 프로바이더로 자동 폴백 (사용자 동의 옵션)

**예외 흐름**
- **E1. 네트워크 오류**: **Gemma 4 온디바이스 모델로 자동 폴백** 분석. 결과 카드에 "오프라인 분석 (정확도 제한)" 표기 + 온라인 복귀 시 본 모델 재분석 옵션 제공
- **E2. LLM 응답 지연 (>10초)**: 진행 표시 후 타임아웃 시 재시도
- **E3. 모호 응답**: 폴백 문구("판단이 어려워요. 가까운 시각의 수유/기저귀를 먼저 확인해보세요") + 사용자 직접 라벨링 옵션

**프라이버시·비용**
- 외부 LLM 전송에 대한 **명시적 동의 플로우** (최초 1회)
- PII 마스킹 (아기 실명 → 가명 토큰)
- 동일 컨텍스트 응답 캐싱으로 비용 절감
- 원본 오디오는 분석 직후 폐기 (사용자 동의 시 학습 개선용 단기 보관)

## UC-1.2 울음 패턴 히스토리 조회

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 누적 울음 분석 결과로 아기 울음 패턴 이해 |

**주 시나리오**
1. **[울음 기록]** 탭 진입
2. 시간대별/카테고리별 통계 차트 — 예) "최근 7일간 17~19시에 '배고픔' 울음 집중"
3. **[패턴 분석]** → LLM이 누적 결과 + 일지로 인사이트 생성
   - 예) "저녁 수유를 30분 앞당겨보는 것을 추천드려요. 최근 5일 중 4일이 18시 직전에 배고픔 울음으로 시작됐어요."
4. 생성된 인사이트는 [UC-4.3 AI 맞춤 팁](../UC-4-baby-info-settings/README.md#uc-43-ai-맞춤-팁-제공)에 통합되어 홈 화면에 표시됨

---

## 비기능·운영 사항

- **녹음 조건**: 최소 10초 이상 (정확도 확보 위해 사전 검증)
- **오프라인 정책**: Gemma 4 폴백으로 동작 가능 (단, 정확도 명시)
- **응답 시간 목표**: 5~10초 이내 (지연 시 진행률 표시)
