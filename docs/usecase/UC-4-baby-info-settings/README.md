# UC-4. 아기 정보 및 설정

> 아기 프로필, 성장 기록, AI 맞춤 팁, AI 프로바이더(LLM) 선택과 관리를 다룬다.

상위: [메인 시나리오](../main-scenario.md)

---

## UC-4.1 아기 프로필 등록

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 아기 기본 정보 등록으로 맞춤 서비스 활성화 |
| 사전조건 | 회원가입 완료 |

**입력 항목**
- 필수: 이름(또는 태명), 성별, 생년월일
- 선택: 출생 시 체중/신장, 출생 주수(조산 여부), 알러지/특이사항, 프로필 사진
- 다중 등록: 쌍둥이/형제 다중 프로필 지원

**주 시나리오**
1. 최초 실행 시 또는 **[설정] → [아기 추가]**
2. 정보 입력 → 저장
3. 등록 정보 기반 홈 화면 개인화 — 생후 일수, 또래 평균 대비 성장 그래프 등

## UC-4.2 성장 기록 · 표준 성장 곡선 · 예방접종 · 이른둥이 모드

**주 시나리오**
1. **[성장]** 메뉴에서 정기적으로 체중/신장/머리둘레 입력
2. **WHO 성장 곡선 + 한국 질병관리청 소아청소년 성장도표** 위에 자동 플롯
3. 또래 백분위수 표시 (3·15·50·85·97 백분위 라인)
4. 예방접종 일정 자동 등록 및 알림 (국가 필수 + 선택 접종)

**이른둥이 모드 (Preemie Mode)** ⭐ NEW
- 출생 주수 < 37주 (조산아) 등록 시 자동 활성화
- **출산예정일 기준 보정 연령**(corrected age)으로 표준 성장 곡선 비교
- 실제 생후 일수와 보정 연령을 함께 표시 ("생후 90일 / 보정 60일")
- 시장 검증: 베이비타임 등 한국 1위 앱 표준 기능, **우리는 필수 진입 요건**

**성장 분석 보고서** ⭐ NEW
- 입력 누적 데이터 기반 정기 보고서 (월 1회 / 분기 1회)
- 우리 아기 vs **같은 연령대 또래 비교**: 키·체중·머리둘레 백분위 추이
- 발달 속도 라인 (정상·관찰 권장·의료 상담 권장 3단계)
- LLM 기반 한 줄 코멘트 (의료 면책 문구 자동 삽입)
- Pro 사용자는 PDF 내보내기 + 가족 공유 우선

**부가**
- 측정 알림 (주기적 입력 권장 — 0~6개월 월 1회, 6~24개월 분기 1회)
- 성장 곡선·보고서 가족 그룹 공유
- 단위 표시: cm/in, kg/lb 토글 (UC-2와 일관)

**경쟁 정렬**: 베이비타임(누적 400만, 4.8★)이 위 모든 기능을 무료 제공. **우리는 베이비타임의 풀 기능 + AI 예측·울음·음성 + 사진/영상 무제한**으로 차별. UC-4.2는 시장 진입 필수 요건.

## UC-4.3 AI 맞춤 팁 제공

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 아기 정보 + 누적 데이터 기반의 시의적절한 조언 |

**팁 제공 트리거**
| 유형 | 예시 |
|---|---|
| 발달 단계 기반 | 생후 3개월: "이 시기에는 목 가누기를 시작합니다. 터미 타임을 늘려보세요" |
| 기록 패턴 기반 | "최근 3일간 수유량이 평소의 70%로 감소했습니다. 컨디션을 확인해보세요" |
| 계절/날씨 기반 | "오늘 미세먼지 나쁨, 실내 활동을 권장드려요" |
| 예방접종 기반 | "다음 주 화요일 DTP 2차 접종 예정입니다" |
| 울음 분석 기반 | "최근 '불편함' 울음이 증가했어요. 기저귀 발진을 확인해보세요" |

**주 시나리오**
1. 홈 화면 상단 **오늘의 팁** 카드 표시
2. 카드 탭 → 상세 설명 확인
3. **[유용했어요/별로였어요]** 피드백으로 추천 정확도 개선
4. **[관련 기록 보기]** 로 근거 데이터 확인

**팁 카테고리**
- 수유/이유식 가이드
- 수면 교육 팁
- 발달 단계별 놀이 추천
- 건강 관리 (예방접종, 증상)
- 부모 셀프케어

**월령별 발달 체크 (Milestone Check)** ⭐ NEW
- 0~24개월 핵심 발달 이정표 체크리스트 (예: 3개월 목 가누기, 6개월 뒤집기, 9개월 앉기, 12개월 첫 걸음·첫 단어)
- 사용자가 월령별 ✓/❌/지연 체크 → 누적 데이터 기반 **발달 지체 가능성 조기 발견** 알림
- LLM 기반 해석 + 의료 면책: "전문가 상담을 권장드려요" (3개 이상 지연 항목 시)
- 시장 검증: 베이비타임 표준 기능, **우리는 시장 진입 필수**
- AI 차별 영역: 누적 일지·울음 패턴과 통합 분석 (베이비타임은 단순 체크리스트만)

**주의사항**
- 모든 AI 팁에 "참고용이며 의학적 진단이 아닙니다" 면책 문구
- 응급 증상 키워드 감지 시 → 즉시 의료기관 방문 안내 배너

## UC-4.4 AI 프로바이더 선택 및 관리

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 종합 추론·AI 팁에 사용할 **텍스트 LLM** 프로바이더 선택·전환 |
| 사전조건 | 앱 로그인. (BYOK/Gemini 연동 시) 해당 계정·키 |

> **범위 주의**: UC-4.4는 **텍스트 LLM 프로바이더**만 다룬다. UC-1의 Tier 1 음향 분류기는 사용자가 선택하는 대상이 아니라 자체 학습·관리되는 온디바이스 모델이며, 별도 OTA 갱신 경로로 배포된다 ([UC-1 training-plan.md](../UC-1-cry-analysis/training-plan.md)). 따라서 본 화면에는 노출되지 않는다.

**지원 프로바이더 (모두 텍스트 호출)**
| 구분 | 프로바이더 | 모델 | 비고 |
|---|---|---|---|
| **기본 (운영사 키)** | Google AI Studio | Gemini 2.5 Flash | 신규 사용자 즉시 사용, 호출비 운영사 부담 (사용량 정책) |
| **Gemini 유료 연동** | Google 계정 OAuth | **최신 Gemini 모델** | Google AI Pro/Ultra 등 구독자가 본인 계정으로 최신 모델 사용 |
| **BYOK** | OpenAI | GPT 텍스트 모델 (예: 4o, 4.1) | 사용자 API 키. **오디오 지원 모델 불필요** |
| **BYOK** | Anthropic | Claude 텍스트 모델 | 사용자 API 키. **오디오 지원 모델 불필요** |
| **LLM 폴백** | 온디바이스 또는 룰베이스 | 소형 텍스트 LM | 네트워크 미연결/실패 시 텍스트 LLM 대체. **멀티모달 Gemma 불필요** |

**주 시나리오 — 기본 사용**
1. 신규 사용자는 별도 설정 없이 **Gemini 2.5 Flash 기본 프로바이더**로 즉시 사용
2. 호출 비용은 운영사가 부담 (월간 사용량 한도 적용)
3. 호출 시 응답 카드 하단에 활성 프로바이더 표기 — 예) "분석: Gemini 2.5 Flash"

**주 시나리오 — Gemini 유료 연동 (최신 모델 사용)**
1. **[설정] → [AI 프로바이더] → [Gemini 계정 연결]**
2. Google OAuth로 본인 계정 로그인
3. 앱이 구독 상태 확인 (Google AI Pro/Ultra 등 유료 플랜 여부)
4. 유료 구독자인 경우 **최신 Gemini 모델**(예: Gemini 2.5 Pro, 또는 출시되는 최신 버전) 선택 활성화
5. 이후 모든 LLM 호출은 사용자의 Gemini 구독을 통해 처리

**주 시나리오 — BYOK 전환**
1. **[설정] → [AI 프로바이더]** 진입
2. 프로바이더 선택 (Google AI Studio / OpenAI / Anthropic)
3. API 키 입력
   - 디바이스 키체인/안드로이드 키스토어에 **암호화 저장**
   - 서버에는 평문 저장 금지
4. **[연결 테스트]** → 짧은 더미 호출로 키 유효성 검증
5. 저장 → 이후 호출이 선택한 프로바이더로 라우팅
6. 언제든 **[기본값으로 복원]** 으로 운영사 Gemini로 복귀

**주 시나리오 — LLM 폴백 (네트워크/프로바이더 실패)**
1. 네트워크 미연결 또는 활성 프로바이더 호출 실패 시 자동으로 **온디바이스 텍스트 LM 또는 룰베이스**로 전환
2. 결과 카드에 "오프라인 추론 (요약 한정)" 표기
3. 온라인 복귀 시 **[본 모델로 재분석]** 옵션 표시
4. UC-1의 경우, 폴백 텍스트 LM조차 사용 불가하면 **Tier 1 분류기 결과만으로 결과 카드 표시** ([UC-1 pipeline.md F3](../UC-1-cry-analysis/pipeline.md))

**구현 요구사항 (아키텍처)**
- **프로바이더 추상화 레이어**: 텍스트 LLM 호출부는 통일된 인터페이스. 각 어댑터가 다음 처리
  - 인증 헤더 및 엔드포인트 (또는 OS 권한 - 온디바이스)
  - 텍스트 페이로드 직렬화 ([UC-1 pipeline.md §3](../UC-1-cry-analysis/pipeline.md)의 payload 스키마)
  - 응답 파싱 → 공통 구조화 스키마(원인 후보 + 추론근거 + 권장행동)로 정규화
- **공통 프롬프트 템플릿**: 프로바이더 교체와 무관한 시스템 프롬프트·역할·출력 스키마
- **메트릭**: 프로바이더별 성공률·평균 지연·토큰 사용량 수집
- **오디오 인코딩 책임 없음**: UC-1 Tier 1 분류기에서 처리하므로 LLM 어댑터는 텍스트만 다룬다

**대안 흐름**
- **A1. 운영사 키 무료 한도 초과**: BYOK 또는 Gemini 유료 연동 안내, 다음 일 재시도 안내
- **A2. BYOK 키 무효/만료**: 기본 운영사 프로바이더로 자동 폴백 + 키 갱신 알림
- **A3. Gemini 구독 만료**: 기본(Gemini 2.5 Flash)로 자동 강등, 알림 표시

**예외 흐름**
- **E1. 모든 온라인 텍스트 LLM 실패**: 온디바이스 텍스트 LM 또는 룰베이스로 폴백. UC-1의 경우 분류기 단독 결과 표시로 추가 폴백 ([UC-1 pipeline.md F3](../UC-1-cry-analysis/pipeline.md))
- **E2. 폴백 텍스트 LM 미설치**: "오프라인 추론 모델 다운로드 필요" 안내, Wi-Fi 환경에서 백그라운드 다운로드. UC-1은 분류기 결과만으로 계속 동작 가능

**프라이버시 고려**
- BYOK: 키는 디바이스에만 저장, 서버 전송 금지
- Gemini 유료 연동: OAuth 토큰만 보관, 비밀번호 미저장
- **오디오 비전송**: UC-1의 오디오는 디바이스를 벗어나지 않는다 (Tier 1 분류기 온디바이스). LLM 프로바이더에는 분류기 라벨 + 일지 텍스트만 전송 → 사용자 본인의 BYOK 키로 호출해도 음성이 외부로 나가지 않음 (프라이버시 강점)
