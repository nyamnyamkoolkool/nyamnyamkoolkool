# UC-4. 아기 정보 및 설정

> 아기 프로필, 성장 기록, AI 맞춤 팁, AI 프로바이더(LLM) 선택과 관리를 다룬다.

상위: [메인 시나리오](../main-scenario.md)

---

## UC-4.1 아기 프로필 등록

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 아기 기본 정보 등록으로 맞춤 서비스 활성화 |
| 사전조건 | 회원가입 완료 |

**입력 항목**
- 필수: 이름(또는 태명), 성별, 생년월일
- 선택: 출생 시 체중/신장, 출생 주수(조산 여부), 알러지/특이사항, 프로필 사진
- 다중 등록: 쌍둥이/형제 다중 프로필 지원

**주 시나리오**
1. 최초 실행 시 또는 **[설정] → [아기 추가]**
2. 정보 입력 → 저장
3. 등록 정보 기반 홈 화면 개인화 — 생후 일수, 또래 평균 대비 성장 그래프 등

---

## UC-4.2 성장 기록

**주 시나리오**
1. **[성장]** 메뉴에서 정기적으로 체중/신장/머리둘레 입력
2. **WHO 성장 곡선** 위에 자동 플롯
3. 또래 백분위수 표시
4. 예방접종 일정 자동 등록 및 알림

**부가**
- 측정 알림 (주기적 입력 권장)
- 성장 곡선 가족 그룹 공유

## UC-4.3 AI 맞춤 팁 제공

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 아기 정보 + 누적 데이터 기반의 시의적절한 조언 |

**팁 제공 트리거**
| 유형 | 예시 |
|---|---|
| 발달 단계 기반 | 생후 3개월: "이 시기에는 목 가누기를 시작합니다. 터미 타임을 늘려보세요" |
| 기록 패턴 기반 | "최근 3일간 수유량이 평소의 70%로 감소했습니다. 컨디션을 확인해보세요" |
| 계절/날씨 기반 | "오늘 미세먼지 나쁨, 실내 활동을 권장드려요" |
| 예방접종 기반 | "다음 주 화요일 DTP 2차 접종 예정입니다" |
| 울음 분석 기반 | "최근 '불편함' 울음이 증가했어요. 기저귀 발진을 확인해보세요" |

**주 시나리오**
1. 홈 화면 상단 **오늘의 팁** 카드 표시
2. 카드 탭 → 상세 설명 확인
3. **[유용했어요/별로였어요]** 피드백으로 추천 정확도 개선
4. **[관련 기록 보기]** 로 근거 데이터 확인

**팁 카테고리**
- 수유/이유식 가이드
- 수면 교육 팁
- 발달 단계별 놀이 추천
- 건강 관리 (예방접종, 증상)
- 부모 셀프케어

**주의사항**
- 모든 AI 팁에 "참고용이며 의학적 진단이 아닙니다" 면책 문구
- 응급 증상 키워드 감지 시 → 즉시 의료기관 방문 안내 배너

## UC-4.4 AI 프로바이더 선택 및 관리

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 울음 분석/AI 팁에 사용할 LLM 프로바이더 선택·전환 |
| 사전조건 | 앱 로그인. (BYOK/Gemini 연동 시) 해당 계정·키 |

**지원 프로바이더**
| 구분 | 프로바이더 | 모델 | 비고 |
|---|---|---|---|
| **기본 (운영사 키)** | Google AI Studio | Gemini 2.5 Flash | 신규 사용자 즉시 사용, 호출비 운영사 부담 (사용량 정책) |
| **Gemini 유료 연동** | Google 계정 OAuth | **최신 Gemini 모델** | Google AI Pro/Ultra 등 구독자가 본인 계정으로 최신 모델 사용 |
| **BYOK** | OpenAI | GPT-4o(audio) 계열 | 사용자 API 키 |
| **BYOK** | Anthropic | Claude (audio 지원 모델) | 사용자 API 키 |
| **오프라인 폴백** | 온디바이스 | **Gemma 4** | 네트워크 미연결/실패 시 자동 전환 |

**주 시나리오 — 기본 사용**
1. 신규 사용자는 별도 설정 없이 **Gemini 2.5 Flash 기본 프로바이더**로 즉시 사용
2. 호출 비용은 운영사가 부담 (월간 사용량 한도 적용)
3. 호출 시 응답 카드 하단에 활성 프로바이더 표기 — 예) "분석: Gemini 2.5 Flash"

**주 시나리오 — Gemini 유료 연동 (최신 모델 사용)**
1. **[설정] → [AI 프로바이더] → [Gemini 계정 연결]**
2. Google OAuth로 본인 계정 로그인
3. 앱이 구독 상태 확인 (Google AI Pro/Ultra 등 유료 플랜 여부)
4. 유료 구독자인 경우 **최신 Gemini 모델**(예: Gemini 2.5 Pro, 또는 출시되는 최신 버전) 선택 활성화
5. 이후 모든 LLM 호출은 사용자의 Gemini 구독을 통해 처리

**주 시나리오 — BYOK 전환**
1. **[설정] → [AI 프로바이더]** 진입
2. 프로바이더 선택 (Google AI Studio / OpenAI / Anthropic)
3. API 키 입력
   - 디바이스 키체인/안드로이드 키스토어에 **암호화 저장**
   - 서버에는 평문 저장 금지
4. **[연결 테스트]** → 짧은 더미 호출로 키 유효성 검증
5. 저장 → 이후 호출이 선택한 프로바이더로 라우팅
6. 언제든 **[기본값으로 복원]** 으로 운영사 Gemini로 복귀

**주 시나리오 — 오프라인 폴백 (Gemma 4)**
1. 네트워크 미연결 또는 활성 프로바이더 호출 실패 시 자동으로 **Gemma 4 온디바이스 모델**로 전환
2. 결과 카드에 "오프라인 분석 (정확도 제한)" 표기
3. 온라인 복귀 시 **[본 모델로 재분석]** 옵션 표시
4. Gemma 4 모델 파일은 최초 실행 시 다운로드 또는 앱 번들에 포함 (모바일 GPU/NPU 활용)

**구현 요구사항 (아키텍처)**
- **프로바이더 추상화 레이어**: LLM 호출부는 통일된 인터페이스. 각 어댑터가 다음 처리
  - 오디오 인코딩/업로드 방식 (Gemini inline base64/File API, OpenAI 오디오 input, Claude 오디오 input, Gemma 4 로컬 호출)
  - 인증 헤더 및 엔드포인트 (또는 OS 권한 - 온디바이스)
  - 응답 파싱 → 공통 구조화 스키마(원인 후보 + 추론근거 + 권장행동)로 정규화
- **공통 프롬프트 템플릿**: 프로바이더 교체와 무관한 시스템 프롬프트·역할·출력 스키마
- **메트릭**: 프로바이더별 성공률·평균 지연·토큰/오디오 사용량 수집

**대안 흐름**
- **A1. 운영사 키 무료 한도 초과**: BYOK 또는 Gemini 유료 연동 안내, 다음 일 재시도 안내
- **A2. BYOK 키 무효/만료**: 기본 운영사 프로바이더로 자동 폴백 + 키 갱신 알림
- **A3. Gemini 구독 만료**: 기본(Gemini 2.5 Flash)로 자동 강등, 알림 표시

**예외 흐름**
- **E1. 모든 온라인 프로바이더 실패**: **Gemma 4 폴백**으로 분석 수행. 오프라인 큐에도 저장하여 온라인 복귀 시 재분석 가능
- **E2. Gemma 4 모델 다운로드 미완료**: "오프라인 분석 모델 다운로드 필요" 안내, Wi-Fi 환경에서 백그라운드 다운로드

**프라이버시 고려**
- BYOK: 키는 디바이스에만 저장, 서버 전송 금지
- Gemini 유료 연동: OAuth 토큰만 보관, 비밀번호 미저장
- Gemma 4: 모든 추론이 디바이스 내부에서 처리 → 오디오 외부 전송 없음 (프라이버시 강점)
