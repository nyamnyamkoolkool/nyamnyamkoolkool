## 1. 앱 개요

### 1.1 제품 소개
**냠냠쿨쿨**은 0~24개월 영아를 양육하는 보호자를 위한 AI 기반 통합 육아 보조 앱이다.
울음소리 분석, 육아일지, 가족 공유 사진 달력, 맞춤형 AI 팁을 제공하여 초보 부모의 육아 부담을 경감하고 가족 구성원이 함께 양육에 참여할 수 있는 환경을 만든다.

### 1.2 타겟 사용자
- **주 사용자**: 0~24개월 영아의 보호자 (특히 초보 부모)
- **부 사용자**: 조부모, 배우자, 베이비시터 등 양육에 참여하는 가족 구성원

### 1.3 핵심 가치
| 가치 | 설명 |
|---|---|
| 즉시성 | 울음의 원인을 빠르게 판별하여 대응 시간 단축 |
| 연속성 | 누적된 육아 기록으로 아기의 패턴 파악 |
| 연결성 | 가족이 같은 정보를 실시간으로 공유 |
| 개인화 | 아기별 데이터에 기반한 맞춤 조언 |

## 2. 사용자 페르소나

### 페르소나 A — 지영 (32세, 신생아 엄마)
- 첫 아이 출산 3주차, 육아휴직 중
- 울음의 의미를 몰라 매번 당황함
- 핵심 니즈: **울음 원인 빠른 파악 + 자동 기록**

### 페르소나 B — 민수 (35세, 직장인 아빠)
- 평일 낮에는 아기와 떨어져 있음
- 핵심 니즈: **하루 요약 + 사진 공유**

### 페르소나 C — 영자 (62세, 외할머니)
- 평일 낮 아기 돌봄 담당, 복잡한 앱 부담
- 핵심 니즈: **간단한 기록 입력 + 부모와 실시간 공유**

## 3. 4대 UseCase 큰 틀

### UC-1. 울음소리 AI 분석
**2-tier 파이프라인**: 자체 학습 음향 분류기(온디바이스) → 텍스트 LLM(종합 추론). 오디오는 LLM에 전송하지 않는다. 자세한 설계 결정 배경과 처리 흐름은 [UC-1 pipeline.md](./UC-1-cry-analysis/pipeline.md) · [training-plan.md](./UC-1-cry-analysis/training-plan.md) 참조.
- **UC-1.0** 기술 가설 검증 (Pre-MVP, 1순위 적중률 ≥ 60% 게이트)
- **UC-1.1** 실시간 울음 분석 (녹음 → 분류기 → 텍스트 LLM)
- **UC-1.2** 울음 패턴 히스토리 및 LLM 인사이트
- (AI 프로바이더 선택/관리는 → UC-4에서 다룬다)

### UC-2. 육아일지 (수유 · 수면 · 배변)
액티브 타이머 인터랙션 지양, **사후 시각 입력 + 음성비서 호출** 우선.
- **UC-2.1** 수유 기록
- **UC-2.2** 수면 기록
- **UC-2.3** 배변 기록
- **UC-2.4** 통합 일지 조회 (타임라인/요약/공유)
- **UC-2.5** 음성비서 핸즈프리 기록 (Siri / Google Assistant / Bixby)

### UC-3. 가족 초대 사진 갤러리 (달력형)
- **UC-3.1** 사진 업로드 및 일자별(달력) 정리
- **UC-3.2** 가족 초대 및 권한 관리 (관리자/보호자/공동양육자/뷰어 — 4단계, 친족 관계 14종)
- **UC-3.3** 추억 되돌아보기 (1년 전 오늘, 마일스톤 모음)

### UC-4. 아기 정보 및 설정
- **UC-4.1** 아기 프로필 등록 (다중 아기 지원)
- **UC-4.2** 성장 기록 (WHO 곡선, 예방접종)
- **UC-4.3** AI 맞춤 팁 (발달/패턴/계절/접종/울음 기반)
- **UC-4.4** AI 프로바이더 선택 및 관리 (Gemini/OpenAI/Anthropic/오프라인)

## 4. 핵심 설계 원칙

### 4.1 액티브 인터랙션 최소화
영아 양육 중 양손이 자유롭지 않다. 행동 중 타이머/시작버튼을 누르는 인터랙션은 채택하지 않는다. **사후 시각 입력**과 **음성비서 호출**이 기본.

### 4.2 텍스트 LLM 프로바이더 + 온디바이스 분류기

UC-1은 오디오를 **자체 학습 음향 분류기(온디바이스)**가 처리하고, **종합 추론 LLM**은 텍스트(분류기 라벨 + 일지 컨텍스트)만 받는다. 따라서 LLM 프로바이더는 모두 **텍스트 호출**만 사용한다 — 오디오 지원 여부와 무관하게 동등 선택 가능.

| 구분 | 프로바이더 | 모델 | 비고 |
|---|---|---|---|
| **기본 (운영사 키)** | Google AI Studio | Gemini 2.5 Flash | 신규 사용자 즉시 사용, 비용/속도 균형 |
| **Gemini 유료 연동** | Google 계정 OAuth | **최신 Gemini 모델** | Google AI Pro/Ultra 등 유료 구독자가 본인 계정으로 최신 모델 사용 |
| **BYOK** | OpenAI | GPT 텍스트 모델 | 사용자 API 키 입력 |
| **BYOK** | Anthropic | Claude 텍스트 모델 | 사용자 API 키 입력 (오디오 지원 모델 불필요) |
| **LLM 폴백** | 온디바이스 텍스트 모델 또는 룰베이스 | 소형 텍스트 LM | 네트워크 미연결/실패 시 텍스트 LLM 대체 |
| **울음 분류 (UC-1 Tier 1)** | 온디바이스 자체 분류기 | Core ML / TFLite | 오프라인 동작, 텍스트 LLM과 독립 |

- LLM 호출부는 **프로바이더 추상화 레이어**로 설계 (어댑터 패턴, 공통 프롬프트/응답 스키마, 텍스트 페이로드 한정)
- 네트워크 차단 시 **UC-1은 분류기 결과만으로 결과 카드 표시 가능** ([UC-1 pipeline.md F3](./UC-1-cry-analysis/pipeline.md))
- 자세한 흐름은 [UC-4.4](./UC-4-baby-info-settings/README.md#uc-44-ai-프로바이더-선택-및-관리) 참조

### 4.3 가족 단위 공유
모든 기록/사진은 가족 그룹 멤버에게 실시간 동기화. **4단계 권한**(관리자 / 보호자 / 공동양육자 / 뷰어)으로 조부모·시터 등 다양한 양육 참여자가 안전하게 참여. **친족 관계 14종**(엄마·아빠·외할머니·할머니·이모·고모·시터·새부모 등)으로 양가 구분이 자연스럽게 표시되며, **알림 송신 가시성 토글**로 본인 활동을 다른 멤버에게 알릴지 개별 제어. 자세한 매트릭스는 [UC-3.2](./UC-3-photo-gallery/README.md#uc-32-가족-초대-및-권한-관리) 참조.

## 5. 기능 간 연계 시나리오 — "저녁 6시, 아기가 울기 시작했다"

1. **UC-1.1**: 보호자가 울음 분석 → 10초 녹음
   - **Tier 1 (온디바이스 분류기)**: "배고픔 패턴" 라벨 (top1 confidence 62%)
   - **Tier 2 (Gemini 2.5 Flash, 텍스트)**: "음향 신호와 일지 신호(마지막 수유 3시간 전)가 같은 방향. 가능성 높음"
2. **UC-2.4 연계**: 결과 카드에서 마지막 수유/기저귀/수면 기록이 함께 표시됨
3. **UC-2.1 또는 UC-2.5**: 수유 후 "방금 분유 120 먹였어" 음성 기록
4. **UC-3.1**: 수유 후 사진 촬영 → 갤러리에 자동 분류, 가족에게 푸시
5. **UC-4.3**: 다음 날 AI 팁 → "최근 저녁 수유 주기가 안정화되고 있어요"
6. **가족 동기화**: 모든 단계가 배우자/조부모에게 실시간 공유

> 만약 step 1에서 네트워크가 끊겨 있으면? **분류기 결과("배고픔 패턴")만 카드에 표시**되고 일지 종합 추론은 온라인 복귀 시 자동 보강. 멀티모달 폴백 LLM은 필요하지 않다.

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 응답성 | 울음 분석 끝-끝 < 4초 (분류기 < 0.5s + 텍스트 LLM 2~3s), 기록 입력 3탭 이내 또는 음성 한 문장으로 완료 |
| 핸즈프리 | 모든 핵심 기록(수유/수면/배변)은 음성비서로 입력 가능 (**네이티브 채널 한정**) |
| 오프라인 | 일지 기록은 오프라인 가능(로컬 큐). 울음 분석은 **온디바이스 분류기 결과만으로 결과 카드 표시**(네이티브 채널 한정), 일지 종합 추론은 온라인 복귀 시 자동 보강 |
| AI 인프라 | Gemini 2.5 Flash 기본 / Gemini 유료(최신) / OpenAI / Anthropic. 프로바이더 추상화 레이어(**텍스트 전용**), BYOK 지원. UC-1 Tier 1 분류기는 별도 온디바이스 모델 ([UC-1 training-plan.md](./UC-1-cry-analysis/training-plan.md)) |
| 백엔드 스택 | **Firebase Firestore + Authentication + Cloud Functions + FCM** + **Kakao Cloud Object Storage (사진·영상 본체)** (2026-05-21 갱신). 데이터 모델은 [_specs/data-model.md](./_specs/data-model.md) 참조 |
| 녹음 조건 | 울음 분석 오디오는 **최소 10초 이상** 캡처 |
| 보안 | 아기 사진 E2E 암호화, 가족 그룹 외부 차단, API 키는 디바이스 키체인 |
| 접근성 | 큰 글자 모드(조부모 사용 고려), 한 손 조작 가능 UI |
| 다국어 | 1차 한국어, 향후 영어/일본어 확장 |
| 배포 | iOS App Store · Google Play **네이티브 앱**과 **앱인토스(Apps in Toss) 미니앱** 동시 운영. 동일 백엔드·동일 가족 그룹 데이터 공유 |

## 7. 배포 채널 (Distribution Channels)

냠냠쿨쿨은 **두 채널**로 배포한다. 두 채널은 **동일한 백엔드·동일한 가족 그룹**을 공유하지만, 기능 가용성에 차이가 있다.

### 7.1 네이티브 모바일 앱 (iOS / Android)
- 1차 채널. 모든 기능을 풀 스펙으로 제공.
- 음성비서(Siri/Google Assistant/Bixby), Gemma 4 온디바이스 폴백, 백그라운드 알림, 핸즈프리 기록 등이 모두 동작.

### 7.2 앱인토스 미니앱 (Apps in Toss)
> 참고: <https://github.com/toss/apps-in-toss-examples>, <https://developers-apps-in-toss.toss.im/>

토스 앱 내부에서 실행되는 **웹 기반 미니앱**으로, 토스 사용자가 별도 설치 없이 즉시 진입할 수 있는 **저진입장벽 채널**이다. 신규 사용자 획득과 가족 구성원(특히 조부모) 초대 수락 시 활용한다.

**채택 이유**
- 토스 사용자 풀에서 즉시 진입 가능 → 신규 보호자 획득 비용 절감
- 가족 초대 링크 수락 시 앱 설치 없이 바로 사진/일지 조회 가능 → 조부모/뷰어 onboarding 마찰 감소
- 토스 계정 인증으로 가입 플로우 단축

**플랫폼 능력 (예시 저장소 기준 추정)**
| 능력 | 상태 | 비고 |
|---|---|---|
| 웹 기반 미니앱 (React/Vue/jQuery) | ✅ | `create-ait-app` 스캐폴딩 |
| 토스 계정 로그인 (`with-app-login`) | ✅ | OAuth 유사 흐름 |
| 카메라 권한 (`with-camera`) | ✅ | 사진 업로드 가능 |
| 사진/스토리지 접근 (`with-storage`) | ✅ | 갤러리 업로드 |
| 인앱 결제 (`with-in-app-purchase`) | ✅ | 향후 구독 결제 채널 |
| 위치/연락처 (`with-location-*`, `with-contacts`) | ✅ | 사용 안 함 또는 가족 초대 viral에 활용 |
| 햅틱·네트워크 상태 (`with-haptic-feedback`, `with-network-status`) | ✅ |  |

**전략 명문화**:
- 앱인토스 = **체험 채널** (육아일지 + AI 울음분석 + AI 팁 + 아기 정보 등록)
- 네이티브 = **풀 기능 채널** (사진 갤러리 + 가족 영상 공유 + 음성비서 + 오프라인 분류기 + BYOK)
- 토스에서 사진 갤러리 진입 시도 → "사진은 네이티브 앱에서 가족과 함께 보세요" CTA + 설치 링크
- 가족 초대 수락은 토스에서도 가능 (onboarding 마찰 최소화) — 일지는 즉시 공유, 사진은 네이티브 설치 후

**채널 전환 정책**
- 앱인토스에서 음성비서·오프라인 분류기 단독 응답이 필요한 시점에 "네이티브 앱 설치" CTA 표시 (강제 X).
- 가족 그룹/계정은 채널 간 공유 — 앱인토스에서 가입한 사용자가 네이티브 앱을 설치하면 동일 계정으로 로그인.
- 인증 방식: 앱인토스는 **토스 OAuth** 우선, 네이티브는 Apple/Google/Kakao OAuth. 동일 사용자에 대해 여러 `AuthIdentity` 연결을 허용.

**미해결 사항 (Open Questions)**
- 앱인토스 SDK의 정확한 마이크 권한 / 오디오 녹음 지속시간 제약 (UC-1.1 분류기 입력용 5~30초 녹음 가능 여부) — **개발자센터 문서 추가 확인 필요**
- 앱인토스에서 분류기 추론: 서버 사이드 호스팅 비용·지연 (Cloud Run / Vertex AI / Kakao Cloud GPU 비교 → [UC-1 pipeline.md §5](./UC-1-cry-analysis/pipeline.md), [_specs/cloud-cost-comparison.md](./_specs/cloud-cost-comparison.md) 갱신 사항)
- 토스 푸시 채널의 가족 활동 알림 전달 가능 여부 (가족 신규 일지/사진 알림)
- 토스 인앱 결제와 BYOK·Gemini 유료 연동의 결제 정책 충돌 여부
- 앱 크기/번들 한계 (특히 사진 갤러리 캐시 정책, **분류기 모델 OTA 다운로드 정책**)
- **토스 미니앱(WebView) 환경에서 Firebase Web SDK의 IndexedDB persistence 동작 확인 필요** — 오프라인 캐시 가용성이 채널별로 다를 수 있음
- **앱인토스 OAuth → Firebase Auth Custom Token 발급 흐름** 검증 (Cloud Function 경유)

## 9. 용어 정의

| 용어 | 정의 |
|---|---|
| 보호자 | 아기를 주로 양육하는 부모 또는 법적 보호자 |
| 가족 그룹 | 한 명 이상의 아기를 중심으로 구성된 양육 참여자 집단 |
| 마일스톤 | 첫 미소, 뒤집기, 첫 걸음 등 발달 이정표 |
| BYOK | Bring Your Own Key. 사용자가 본인의 LLM API 키를 직접 등록해 사용하는 방식 |
| 온디바이스 | LLM/모델을 디바이스 내부에서 실행해 외부 네트워크 호출 없이 추론하는 방식 |
| 앱인토스 (Apps in Toss) | 토스 앱 내부에서 실행되는 웹 기반 미니앱 플랫폼. `create-ait-app`으로 스캐폴딩하며 토스 계정·결제·디바이스 권한 일부를 사용 가능 |
| 미니앱 | 모기업 앱(토스 등) 내부에서 실행되는 경량 웹 기반 앱. 별도 설치 없이 즉시 진입 가능 |
| 네이티브 채널 | iOS App Store / Google Play를 통해 배포되는 네이티브 모바일 앱 |
| Firestore | Firebase의 NoSQL 문서 DB. 본 프로젝트의 1차 백엔드 데이터 저장소 |
| Cloud Functions | Firebase의 서버리스 함수 실행 환경. 트리거(onCreate/onWrite)·HTTPS callable·스케줄러 지원 |
| Security Rules | Firestore 접근 제어 언어. Postgres RLS의 Firebase 대응 개념 |
