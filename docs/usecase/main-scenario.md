## 1. 앱 개요

### 1.1 제품 소개
**냠냠쿨쿨**은 0~24개월 영아를 양육하는 보호자를 위한 AI 기반 통합 육아 보조 앱이다.
울음소리 분석, 육아일지, 가족 공유 사진 달력, 맞춤형 AI 팁을 제공하여 초보 부모의 육아 부담을 경감하고 가족 구성원이 함께 양육에 참여할 수 있는 환경을 만든다.

### 1.2 타겟 사용자
- **주 사용자**: 0~24개월 영아의 보호자 (특히 초보 부모)
- **부 사용자**: 조부모, 배우자, 베이비시터 등 양육에 참여하는 가족 구성원

### 1.3 핵심 가치
| 가치 | 설명 |
|---|---|
| 즉시성 | 울음의 원인을 빠르게 판별하여 대응 시간 단축 |
| 연속성 | 누적된 육아 기록으로 아기의 패턴 파악 |
| 연결성 | 가족이 같은 정보를 실시간으로 공유 |
| 개인화 | 아기별 데이터에 기반한 맞춤 조언 |

## 2. 사용자 페르소나

### 페르소나 A — 지영 (32세, 신생아 엄마)
- 첫 아이 출산 3주차, 육아휴직 중
- 울음의 의미를 몰라 매번 당황함
- 핵심 니즈: **울음 원인 빠른 파악 + 자동 기록**

### 페르소나 B — 민수 (35세, 직장인 아빠)
- 평일 낮에는 아기와 떨어져 있음
- 핵심 니즈: **하루 요약 + 사진 공유**

### 페르소나 C — 영자 (62세, 외할머니)
- 평일 낮 아기 돌봄 담당, 복잡한 앱 부담
- 핵심 니즈: **간단한 기록 입력 + 부모와 실시간 공유**

## 3. 4대 UseCase 큰 틀

### UC-1. 울음소리 AI 분석
멀티모달 LLM 기반으로 아기 울음의 원인을 누적 일지 컨텍스트와 결합해 추론한다.
- **UC-1.1** 실시간 울음 분석 (10초 이상 녹음 → LLM 추론)
- **UC-1.2** 울음 패턴 히스토리 및 LLM 인사이트
- (AI 프로바이더 선택/관리는 → UC-4에서 다룬다)

### UC-2. 육아일지 (수유 · 수면 · 배변)
액티브 타이머 인터랙션 지양, **사후 시각 입력 + 음성비서 호출** 우선.
- **UC-2.1** 수유 기록
- **UC-2.2** 수면 기록
- **UC-2.3** 배변 기록
- **UC-2.4** 통합 일지 조회 (타임라인/요약/공유)
- **UC-2.5** 음성비서 핸즈프리 기록 (Siri / Google Assistant / Bixby)

### UC-3. 가족 초대 사진 갤러리 (달력형)
- **UC-3.1** 사진 업로드 및 일자별(달력) 정리
- **UC-3.2** 가족 초대 및 권한 관리 (관리자/보호자/뷰어)
- **UC-3.3** 추억 되돌아보기 (1년 전 오늘, 마일스톤 모음)

### UC-4. 아기 정보 및 설정
- **UC-4.1** 아기 프로필 등록 (다중 아기 지원)
- **UC-4.2** 성장 기록 (WHO 곡선, 예방접종)
- **UC-4.3** AI 맞춤 팁 (발달/패턴/계절/접종/울음 기반)
- **UC-4.4** AI 프로바이더 선택 및 관리 (Gemini/OpenAI/Anthropic/오프라인)

## 4. 핵심 설계 원칙

### 4.1 액티브 인터랙션 최소화
영아 양육 중 양손이 자유롭지 않다. 행동 중 타이머/시작버튼을 누르는 인터랙션은 채택하지 않는다. **사후 시각 입력**과 **음성비서 호출**이 기본.

### 4.2 멀티 LLM 프로바이더 + 오프라인 폴백
| 구분 | 프로바이더 | 모델 | 비고 |
|---|---|---|---|
| **기본 (운영사 키)** | Google AI Studio | Gemini 2.5 Flash | 신규 사용자 즉시 사용, 비용/속도 균형 |
| **Gemini 유료 연동** | Google 계정 OAuth | **최신 Gemini 모델** | Google AI Pro/Ultra 등 유료 구독자가 본인 계정으로 최신 모델 사용 |
| **BYOK** | OpenAI | GPT-4o (audio) | 사용자 API 키 입력 |
| **BYOK** | Anthropic | Claude (audio 지원 모델) | 사용자 API 키 입력 |
| **오프라인 폴백** | 온디바이스 | **Gemma 4** | 네트워크 미연결/실패 시 자동 전환 |

- LLM 호출부는 **프로바이더 추상화 레이어**로 설계 (어댑터 패턴, 공통 프롬프트/응답 스키마)
- 오프라인 모드 분석 결과에는 "오프라인 분석 (정확도 제한)" 표기, 온라인 복귀 시 본 모델 재분석 옵션
- 자세한 흐름은 [UC-4.4](./UC-4-baby-info-settings/README.md#uc-44-ai-프로바이더-선택-및-관리) 참조

### 4.3 가족 단위 공유
모든 기록/사진은 가족 그룹 멤버에게 실시간 동기화. 권한 분리(관리자/보호자/뷰어)로 조부모 등 부 사용자도 안전하게 참여.

## 5. 기능 간 연계 시나리오 — "저녁 6시, 아기가 울기 시작했다"

1. **UC-1.1**: 보호자가 울음 분석 → 10초 녹음 → Gemini 2.5 Flash 추론 결과 "배고픔 78% (마지막 수유 3시간 전)"
2. **UC-2.4 연계**: 결과 카드에서 마지막 수유/기저귀/수면 기록이 함께 표시됨
3. **UC-2.1 또는 UC-2.5**: 수유 후 "방금 분유 120 먹였어" 음성 기록
4. **UC-3.1**: 수유 후 사진 촬영 → 갤러리에 자동 분류, 가족에게 푸시
5. **UC-4.3**: 다음 날 AI 팁 → "최근 저녁 수유 주기가 안정화되고 있어요"
6. **가족 동기화**: 모든 단계가 배우자/조부모에게 실시간 공유

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 응답성 | 울음 분석 5~10초 이내, 기록 입력 3탭 이내 또는 음성 한 문장으로 완료 |
| 핸즈프리 | 모든 핵심 기록(수유/수면/배변)은 음성비서로 입력 가능 |
| 오프라인 | 일지 기록은 오프라인 가능(로컬 큐). 울음 분석은 **Gemma 4 온디바이스 폴백**으로 오프라인에서도 동작 |
| AI 인프라 | Gemini 2.5 Flash 기본 / Gemini 유료(최신) / OpenAI / Anthropic / Gemma 4. 프로바이더 추상화 레이어, BYOK 지원 |
| 녹음 조건 | 울음 분석 오디오는 **최소 10초 이상** 캡처 |
| 보안 | 아기 사진 E2E 암호화, 가족 그룹 외부 차단, API 키는 디바이스 키체인 |
| 접근성 | 큰 글자 모드(조부모 사용 고려), 한 손 조작 가능 UI |
| 다국어 | 1차 한국어, 향후 영어/일본어 확장 |

## 8. 용어 정의

| 용어 | 정의 |
|---|---|
| 보호자 | 아기를 주로 양육하는 부모 또는 법적 보호자 |
| 가족 그룹 | 한 명 이상의 아기를 중심으로 구성된 양육 참여자 집단 |
| 마일스톤 | 첫 미소, 뒤집기, 첫 걸음 등 발달 이정표 |

| 온디바이스 | LLM/모델을 디바이스 내부에서 실행해 외부 네트워크 호출 없이 추론하는 방식 |
