# UC-2. 육아일지 (수유 · 수면 · 배변)

> 액티브 인터랙션(타이머/시작 버튼) 지양. **사후 시각 입력 + 음성비서 호출**이 기본.

상위: [메인 시나리오](../main-scenario.md)

---

## 설계 원칙

수유/수면 도중에는 양손이 비어있지 않다. 행동 중 앱을 켜고 타이머를 누르는 인터랙션은 채택하지 않는다.
- **사후 입력**: 끝난 뒤 시각을 입력하는 방식이 기본
- **빠른 입력**: 직전 패턴 원터치 복제
- **음성비서**: Siri / Google Assistant / Bixby로 핸즈프리 기록 (UC-2.5)

### 입력 UX 4대 원칙 (사용자 실측 검증 2026-05-20)

쑥쑥찰칵 사용자의 실측 불편 4가지를 직접 해소하는 입력 설계.

| # | 불편 (쑥쑥찰칵) | 우리 해결 메커니즘 | KPI |
|---|---|---|---|
| 1 | 시각 휠 입력이 번거로움 | ① 기본값 = 현재 시각, "확인" 1탭 / ② 상대 시각 칩 "방금/10분 전/30분 전/1시간 전" / ③ 음성 자연어 "12시 30분" | 시각 입력 0~1탭 |
| 2 | 종류·양 선택지가 많아 고름이 부담 | ① 직전 입력 자동 미리 선택 (Smart Default) / ② 자주 쓰는 양 칩 (60·90·120·150·180ml) / ③ 가장 최근 메뉴 1탭 복제 | 종류·양 선택 0~1탭 |
| 3 | 한 손으로 조작 어려움 | ① 모든 핵심 버튼 하단 Thumb Zone 배치 / ② 최소 터치 영역 60dp / ③ 좌·우 스와이프로 종류 전환 / ④ 음성비서 = 0손 | 한 손 조작 100% |
| 4 | 시작/종료를 모두 눌러야 함 (수면) | ① 사후 일괄 입력 ("방금 깼어" 1번에 마감) / ② 잠든 시각 미상 시 누적 패턴 자동 추정 / ③ 진행 중 수면 허용 (`woke_up_at` NULL → PATCH) | 수면 1건 1탭 |

**목표 KPI**:
- 일지 1건 입력 평균 **1~3탭** (쑥쑥찰칵 추정 5~7탭 대비 50%+ 단축)
- 일지 1건 입력 평균 **3~5초** (쑥쑥찰칵 추정 20~30초 대비 80%+ 단축)
- 한 손 조작 가능 비율 **100%**
- 양손이 빈 상태 조작 비율 **100%** (음성비서 사용 시)

### 단위 변환·표시 정책 ⭐ NEW

베이비타임 등 한국 시장 표준 기능. 우리는 시장 진입 필수 요건으로 채택.

| 항목 | 기본 (KR) | 변환 옵션 |
|---|---|---|
| 수유량 | **ml** | oz (1 oz = 29.57 ml) |
| 키·머리둘레 | **cm** | inch (1 in = 2.54 cm) |
| 체중 | **kg** | lb (1 lb = 0.4536 kg) |
| 체온 | **°C** | °F |
| 수면 시간 | 시:분 | hh:mm 또는 분 단위 |

- 사용자 설정 한 번으로 앱 전체 단위 일괄 변경
- 가족 그룹 내 멤버별 단위 설정 분리 가능 (해외 거주 친척 대응)

### 베이비타임 정렬 ⭐ NEW

**시장 진입 필수 동등 기능** (베이비타임 풀 기능):
- ✅ 수유·수면·배변 일지 (사후 입력 + Smart Default)
- ✅ 단위 변환 (ml/oz, cm/in, kg/lb)
- ✅ 사진 함께 저장 (UC-3과 연동)
- ✅ Apple Watch / Wear OS 빠른 입력 (v1.x, UC-2.6 참조)
- ✅ 위젯 (v1.x, 홈 화면 1탭 입력)

**베이비타임 대비 차별 영역** (우리만):
- ⭐ AI 수유·수면 예측 (다음 윈도우 미리 알림, UC-1.2 + UC-4.3)
- ⭐ 음성비서 핸즈프리 (UC-2.5, Siri/Bixby/Google Assistant + Web Speech API)
- ⭐ 가족 8명 무료 (베이비타임은 추가 가족 ₩3,300 일회성)

## UC-2.1 수유 기록

| 항목 | 내용 |
|---|---|
| Actor | 보호자, 가족 구성원 |
| 목표 | 수유 시간/양/종류 기록, 다음 수유 타이밍 예측 |
| 사전조건 | 아기 프로필 생성 완료 |

**주 시나리오 (Smart Default 적용)**
1. **[+ 기록]** → **[수유]** 선택 (또는 홈 화면 "방금 수유" 한 손 1탭)
2. **자동 미리 채워짐**:
   - 종류: 직전 수유와 동일 (예: 분유)
   - 양: 직전 양 (예: 120ml) — 자주 쓰는 양 칩(60·90·120·150·180)으로 즉시 변경 가능
   - 시각: 현재 시각 — 상대 시각 칩("방금"/"10분 전"/"30분 전"/"1시간 전")으로 즉시 변경
3. 변경 없으면 즉시 **[저장]** → 총 **1탭**으로 완료
4. 메모는 선택 (기본 표시 X, 펼치기로만)

**보조 기능 (모두 사후 입력)**
- **홈 화면 위젯**: "방금 수유" 1탭으로 직전 패턴 그대로 기록 (앱 미진입)
- **음성 입력**: "방금 분유 120 먹였어" → UC-2.5
- **다음 수유 예상 시간**: 누적 패턴 기반 홈 화면 자동 표시
- **한 손 조작**: 모든 입력 컨트롤은 화면 하단 1/3 영역 (Thumb Zone)

## UC-2.2 수면 기록

> 잠들 때 버튼을 누르는 것은 비현실적. **깬 뒤 회상 입력 1회**가 기본 — 시작/종료 두 번 누름 회피.

**주 시나리오 (사후 일괄 입력)**
1. 홈 화면 **"방금 깼어"** 1탭 또는 **[+ 기록] → [수면]** 선택
2. **자동 미리 채워짐**:
   - 깬 시각: 현재 시각
   - 잠든 시각: 누적 수면 패턴 기반 자동 추정 (예: "약 2시간 전 19:30 추정")
   - 낮잠/밤잠: 시각 기준 자동 분류
3. **추정값 OK면 즉시 [저장]** → 1탭 완료
4. 추정 틀리면 상대 시각 칩 ("1시간 전 / 2시간 전 / 3시간 전") 또는 음성으로 정정

**진행 중 수면 처리 (`woke_up_at` NULL)**
- 음성 "아기 방금 잠들었어" → SleepLog 생성 (`fell_asleep_at`만, `woke_up_at`은 NULL)
- 깰 때 음성 "방금 깼어" → 동일 `client_event_id`로 PATCH (`woke_up_at` 추가)
- 사용자는 시작 버튼을 절대 누르지 않음 — 깰 때만 1번 입력

**보조 기능**
- **음성 입력**: "아기 12시 반에 잠들어서 2시에 깼어" → 한 문장으로 완료
- 일/주/월 단위 수면 그래프, 또래 평균 비교, 수면 패턴 이상 감지 알림
- 누적 패턴 학습으로 다음 수면 시각 예측 (UC-1.2 패턴 인사이트 활용)

## UC-2.3 배변 기록

**주 시나리오**
1. **[+ 기록]** → **[배변]** 선택
2. 종류: 소변 / 대변 / 혼합
3. 대변의 경우 색상(노랑/녹색/갈색 등), 묽기(묽음/보통/단단함) — 아이콘 기반 선택
4. 메모 (선택, 발진 여부 등)
5. 저장

**알림**
- 장시간 배변 미기록 시 알림
- 색상/상태 이상 패턴 감지 시 "소아과 상담 권장" 안내

## UC-2.4 통합 일지 조회

| 항목 | 내용 |
|---|---|
| Actor | 보호자 |
| 목표 | 하루/기간 단위 모든 기록 종합 |

**주 시나리오**
1. **[일지]** 탭 → 날짜 선택
2. **타임라인 뷰**: 시간순으로 수유/수면/배변/울음 분석 결과 통합 표시
3. **[요약]** 토글 → 일별 통계 카드 (수유 횟수/총 수면시간/배변 횟수)
4. **[공유]** → PDF/이미지로 내보내기 (소아과 방문 시 활용)

## UC-2.5 음성비서 핸즈프리 기록

| 항목 | 내용 |
|---|---|
| Actor | 보호자, 가족 구성원 |
| 목표 | 손이 자유롭지 않은 상황에서 음성으로 기록 |
| 사전조건 | 음성비서 앱 + 냠냠쿨쿨 계정 연동 |

**지원 플랫폼**
- iOS: Siri Shortcuts (App Intents 프레임워크)
- Android: Google Assistant (App Actions / BII)
- Samsung: Bixby Capsule

**주 시나리오**
1. 음성비서 호출
   - "헤이 빅스비, 냠냠쿨쿨에 분유 120 기록해줘"
   - "시리야, 냠냠쿨쿨에서 아기 방금 잠들었다고 기록"
   - "오케이 구글, 냠냠쿨쿨에 기저귀 갈았어"
2. 음성비서가 앱 Intent로 전달
3. 백그라운드 기록 생성 → 음성비서가 결과 음성 확인
   - "분유 120ml 5시 14분에 기록했어요"
4. 신뢰도 낮으면 재확인 질문 ("수유량이 120ml 맞나요?")

**지원 발화 예시**
| 의도 | 발화 예시 | 추출 정보 |
|---|---|---|
| 수유 | "분유 백이십 먹였어" | 종류=분유, 양=120ml, 시각=now |
| 수면 시작 | "아기 방금 잠들었어" | 잠든시각=now |
| 수면 종료 | "8시에 잠들어서 방금 깼어" | 잠든시각=08:00, 깬시각=now |
| 배변 | "기저귀 갈았어, 응가" | 종류=대변, 시각=now |
| 울음 분석 호출 | "냠냠쿨쿨로 울음 분석해줘" | → 앱 열고 UC-1.1 |
| 조회 | "오늘 수유 몇 번 했어?" | → 음성 통계 응답 |

**대안 흐름**
- **A1. 파싱 실패**: "정확히 무엇을 기록할까요?" 재질의
- **A2. 시각 모호 (예: '아까')**: 직전 동일 카테고리 시각 기준 추정 + 사용자 확인
- **A3. 다중 아기**: "어느 아기인가요?" 추가 질의 (이후 컨텍스트 기억)

**예외 흐름**
- **E1. 오프라인**: STT는 동작, 기록은 로컬 큐 → 온라인 복귀 시 동기화
- **E2. 미인증 화자**: 음성 인증 활성 시 가족 외 발화는 무시

**부가 가치**
- 운전 중, 수유 직후 손이 더러울 때, 한밤중 어둠 속에서도 기록 가능
- 위젯/숏컷으로 가장 자주 쓰는 기록 한 탭 실행

## UC-2.6 웨어러블 빠른 입력 (Apple Watch · Wear OS) — v1.x

| 항목 | 내용 |
|---|---|
| Actor | 보호자, 가족 |
| 목표 | 손목에서 1탭 또는 음성으로 핵심 기록 입력 |
| 사전조건 | 페어링된 시계 보유, 컴패니언 앱 설치 |

**플랫폼 범위 (v1.x 검토)**
- **Apple Watch (watchOS)**: SwiftUI 컴패니언 앱 + Siri Watch Face / Smart Stack 통합
- **Wear OS (Android)**: Compose for Wear OS + Google Assistant 연동
- 시장 검증: 베이비타임 Apple Watch 지원이 표준. 우리는 v1.x로 동등 진입 필요.

**핵심 기능**
- 1탭 "방금 수유 / 깼어 / 똥" 즉시 기록 (Smart Default 적용)
- 음성 한 문장 → STT (Watch onboard) → Cloud Function 파싱 → 기록 저장
- 진행 중 수면 상태 표시 (워치 컴플리케이션)
- 가족 알림 (배우자가 기록 추가 시 워치 햅틱)

**버전 결정**
- v1 (3개월 MVP): **미포함** (WebView 하이브리드 우선)
- **v1.x (정식 출시 이후 3~6개월)**: Apple Watch + Wear OS 동시 출시 — 베이비타임 동등 진입 + 핸즈프리 차별 강화
- 동기: 베이비타임의 Apple Watch 호환은 시장 표준이며, 부재 시 평가위원·사용자 모두 "기본 기능 부재"로 인식할 위험

**비기능**
- 워치 단독 동작 (폰 미연결 시 로컬 큐 → 폰 동기화 시 업로드)
- 배터리 최적화: 컴플리케이션 갱신은 push 기반 (polling 금지)
